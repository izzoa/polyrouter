Close four cost-recording completeness gaps so the spend record budgets and dashboards reconcile from is never silently under-counted (FABLE_AUDIT E5).

**Shutdown flush drains to completion (E5.1).** `LogWriter.flush()` early-returned while a flush was in flight, and timer flushes were un-awaited — so if a flush was mid-retry when SIGTERM arrived, the shutdown flush no-oped and every draft enqueued after that flush's splice (including the final rows from just-drained streams) was lost *without* being counted as dropped. Flush now coalesces onto the in-flight promise and a shutdown drain loop runs until both queues are empty; each individual DB op (per-draft price lookup and each batch insert) is bounded by a `opTimeoutMs` deadline so a hung database can't block the drain (its rows are counted-as-dropped, never silent) — and a large healthy batch is never falsely dropped. `flushOnce` snapshots both queues up front so a cascade log and its attempt land in the same cycle.

**A cancelled cascade cheap leg records a row (E5.2).** When the client disconnected during the cascade's buffered cheap attempt, both branches threw without recording — the request vanished from the spend record and inspector. They now record exactly one `status=error` row (cheap tier, `escalated=false`, no `request_attempt` rows) before propagating, and do not emit a provider-failure notification (a client disconnect is breaker-neutral, not a fault).

**§8 BYOK families are priceable (E5.3).** DashScope/Qwen (`dashscope`), Moonshot/Kimi (`moonshot`), MiniMax (`minimax`), and Zhipu/Z.ai GLM (`zai`) had no host→family mapping, so they recorded `cost=null` forever. Added the LiteLLM-verified mappings for their **international (USD) endpoints only** — the China-domestic endpoints (`dashscope.aliyuncs.com`, `api.moonshot.cn`, `api.minimax.chat`, `api.minimaxi.com`, `open.bigmodel.cn`) bill in CNY and are deliberately left unmapped (recorded as unknown, never a currency-wrong USD cost). Extended the bundled catalog with real per-token rows for these families plus the mapped-but-unpriced xai/cohere hosts, and bumped `BUNDLED_CATALOG_VERSION`.

**A stale model price never overrides the catalog for a known provider (E5.4).** `resolveModelPrice` now honors a model-own price only for `custom`/`local` providers; for `api_key`/`subscription` it falls through to the catalog (the API already forbids setting a price on those kinds). So a stale price left after a `custom → api_key` kind change — or one written by a request racing that change — can never produce a wrong cost. The kind change also clears the models' now-ignored prices so `GET /api/models` stays consistent. No schema migration; historical `request_log` cost snapshots are unaffected.
