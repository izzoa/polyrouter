Fix dashboard text contrast to WCAG floors (StyleSeed/design-lock fix #2). The tertiary text token `--text3` was below 4.5:1 on the surfaces it sits on (≈3.3:1 light, and ≈4.4:1 on dark chips) — it is now `#6a6c73` (light) / `#898b93` (dark), verified ≥4.6:1 against background, panel, AND chip in both themes. The whisper tint `--faint` (≈2:1) is reclassified as decorative-only: every meaningful usage — SSRF/encryption help notes, empty-state fallbacks, the drag-to-reorder instruction, layer metadata tags, the provider "Not tested yet" status, unpriced cost values, budget enabled/disabled status, the serving-host line — migrated to the passing `--text3`, while glyphs (`⋮⋮`, `→`, `⧉`, legend keys) keep the whisper. The interactive `×`/`✕` controls and the switch's off-state track also move to the ≥3:1 token (WCAG 1.4.11 — ARIA state doesn't substitute for sighted contrast). The Overview provider strip no longer conveys status by dot color alone (aria-label + title name the status). A new Vitest suite parses the shipped `styles.css` and asserts every text-token/surface pair per theme, plus a pinned allowlist that fails if `color: var(--faint)` ever reappears on meaningful copy.
