Make the dashboard keyboard-operable and screen-reader-honest (StyleSeed/design-lock fix #1). Every interactive control is now a real `<button>` (previously all actions except Login's were click-handling `div`/`span`s — invisible to the keyboard): nav, filters, range/login/threshold segments, table rows, icon `✕`/`×` controls, theme toggle, endpoint chip, and every btn-primary/ghost/cancel action. A global `:focus-visible` accent ring replaces the blanket `outline: none`, so keyboard focus is always visible while pointer interaction stays quiet. Form fields gain real `<label for>` association (no more placeholder-named inputs). The switch control is a native `role="switch"` with `aria-checked`/`aria-disabled` (a locked switch is discoverable but inert — previously only `pointer-events` blocked mice, not keyboards). The toast is a polite `role="status"` live region. The inspector drawer and modals are real dialogs: `role="dialog"` + `aria-modal` backed by focus-in on open, an in-dialog Tab loop, focus restore on close, and Escape closing the topmost overlay deterministically (modal first, drawer second). Busy actions are natively `disabled` instead of pointer-inert. A local eslint rule (`a11y-guard`) rejects future `onClick`-on-div regressions and orphaned labels. Pointer-visible rendering is unchanged apart from the focus ring.
