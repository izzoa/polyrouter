Add the persistence foundation: Drizzle + PostgreSQL 16 schema for the identity/config core (users, organizations, agents, providers, models, tiers, routing entries/rules) with migrations generated by drizzle-kit and applied automatically on boot; a central tenant-scoping guard (every query ownership-scoped, ownership immutable, parent-linked rows tenant-checked at mutation time, raw database handles private to the persistence module); an AES-256-GCM secret-encryption utility; a shared Redis (ioredis) module; a new server-only entrypoint `@polyrouter/shared/server`; and `docker-compose.dev.yml` for local PostgreSQL/Redis. Database-enforced invariants include the five-models-per-tier cap and idempotent default-tier provisioning.
