Make `install.sh` idempotent for a **fetch** install re-run and close two secret-rotation footguns (FABLE_AUDIT epic E13 + backlog A-19). No package version bump — installer/docs/CI only.

- **A fetch-install re-run no longer rotates secrets or nests.** The prior-install check only recognized a working-tree checkout, so re-running the one-liner from **inside** a fetch install (`polyrouter/{src/,.env}`) fell into the fetch branch, nested `polyrouter/polyrouter/`, missed the `.env`, and generated brand-new secrets — booting compose against the existing volumes with a fresh `POSTGRES_PASSWORD` (can't auth an already-initialized DB → crash-loop) and a rotated `PROVIDER_CREDENTIAL_KEY` (orphans every stored credential). A prior fetch install is now recognized by a durable, polyrouter-specific marker (`.polyrouter-install`) and refreshed **in place** — reusing the existing `.env` (byte-identical) and re-fetching `src/` without nesting.
- **Unrelated directories are never adopted.** Detection keys on our marker, not a bare `src/docker-compose.yml` (which any project may have), and the fresh-install path refuses to overwrite a pre-existing unmarked `polyrouter/` directory — so the installer can never replace an unrelated project's `src/`.
- **A deleted `.env` refuses regeneration in both modes.** A `.polyrouter-secrets-created` sentinel records that secrets were generated in a directory; if it is present but `.env` is gone (a fetch install with an interrupted first run, or a checkout whose `.env` was deleted), the installer fails with guidance to restore from backup rather than silently rotating keys against live volumes. The source swap also keeps the previous `src/` until the refreshed tree lands, so an interrupted refresh recovers.
- **A-19:** the README documents that the `docker compose -p polyrouter-selfhost …` commands assume a checkout; a fetch install runs them from inside `polyrouter/` with `--env-file .env -f src/docker-compose.yml --project-directory src` (the manage command the installer prints).

A POSIX shell regression test (`test/install-rerun.test.sh`, wired into CI) stubs `docker`/`curl` and asserts all of the above (20 assertions); it fails against the pre-fix script and passes against the fix.
