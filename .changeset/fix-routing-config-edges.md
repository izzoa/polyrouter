Fix three routing-config edge defects (FABLE_AUDIT epic E10):

- **Explicit JSON `null`s now 4xx, not 500.** `@IsOptional()` skipped validators for `null` as well as `undefined`, so a rule POST/PATCH with `{target:null}` threw a `parseRoutingTarget` TypeError (500), `{priority:null}`/`{matchType:null}` hit a NOT-NULL violation (500), and `{headerName:null}` silently rewrote the rule to the default header. A defined-only `IfDefined()` validator (`@ValidateIf((_o,v)=>v!==undefined)`) now rejects an explicit `null` on every non-nullable field with a clean 4xx and leaves the stored rule unchanged.
- **A cascade-deleted position-0 model no longer bricks a tier.** `resolveTier` requires an exact `position === 0` (no silent fallback promotion), but deleting a provider/model cascaded its `routing_entries` and nothing re-compacted — leaving a tier with healthy models at 1..N returning `empty_tier` until a manual re-save. Provider/model delete now compacts every owner tier's entry positions back to contiguous `0..N-1` inside the delete transaction (ascending renumber, no `(tier_id, position)` unique-index collision), so a surviving fallback still serves. The transaction takes a tier-first `FOR UPDATE` lock so it can't deadlock against a concurrent `replaceForTier`. `resolveTier`'s no-silent-promotion stance is unchanged.
- **The structural baseline can re-learn after a rotating-fingerprint flood.** The per-agent EWMA hash capped at 32 fields and, when full, refused new fingerprints *while refreshing the whole-hash TTL* — so a harness interpolating dynamic values (timestamp, cwd, session id) into the system prompt filled the hash in 32 requests and the shared, cross-instance baseline could never learn legitimate boilerplate again. `EWMA_LUA` now keeps a sibling per-agent LRU ZSET (field → server-clock last-touch) and, at cap, `ZPOPMIN`+`HDEL`s the stalest field before inserting the new one — still one atomic script. The key namespace is versioned (`route:sbaseline:v2:…`) so legacy ZSET-less hashes are ignored and expire on their TTL. No schema change.
