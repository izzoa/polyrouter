Add the notification producers (spec §10.1/§12; invariants 1/5/6/8/10/11) — the events that emit into #15a's delivery core, plus the password-reset system email (the `sendResetPassword` stub #3 left). Every producer is fire-and-forget over the non-blocking `emit`, so no producer/notification/mailer failure or latency touches a proxy request, fallback, budget check, or reset.

- **`provider_down`**: the data-plane circuit breaker's `complete()` now returns `{ justOpened, generation, openedAt }` (additive — the state machine and the Lua HMSET are unchanged; the discarded `1`/`0` Lua return became a tuple on every path). `withBreaker`/`withBreakerStream` gain an optional `onOpen` callback the fallback-chain walkers thread per attempt; `ProxyService` supplies one that emits `provider_down` for the tripped provider (owner = the request principal, provider name from the loaded `ProviderRow`). `justOpened` is surfaced **only for the shared (primary) breaker** — a per-instance in-memory fallback open never alerts — so it's one best-effort emission per open transition, deduped per `(owner, provider)` within the 1h window.
- **`request_failures_spike`**: an atomic Redis Lua counter (`INCR` + first-increment `PEXPIRE`) per owner per epoch-aligned window, incremented off the request path whenever a chain records `status='error'` (every path: buffered, pre/post-commit stream, cascade), emitting once when the count reaches `NOTIFY_FAILURE_THRESHOLD`. No DB query, no async-log-writer race, correct across instances.
- **`weekly_spend_summary`**: a narrow, scheduler-only `WEEKLY_SPEND_READER` (built inside `DatabaseModule` on the private drizzle handle, never re-exported) that sums both cost ledgers — `request_log.cost` + cascade `request_attempt.cost`, null→0 — over a bounded `[start, endExclusive)` window per owner. A BullMQ Job Scheduler (`upsertJobScheduler`, UTC, opt-in via `NOTIFY_WEEKLY_ENABLED`) runs it once per occurrence across instances with an occurrence-keyed idempotent handler; **bootstrap reconciliation is fail-open** so a down Redis can never gate Layer-0 boot (invariant 1). The rendered metric is "Known spend" (unknown-price rows count as 0).
- **Password-reset email**: server-wide `SMTP_*` defaults (§12) power a `SystemMailer` that sends via #15a's SSRF-guarded, connect-time IP-pinned `deliverSmtp`; Better Auth's `sendResetPassword` is rewired to a **detached** send (Better Auth awaits the hook, so the reset request must not wait on SMTP) that never logs the token/url and warns+skips when SMTP is unconfigured.

New config: `SMTP_HOST`/`PORT`/`USER`/`PASS`/`FROM`/`SECURE`, `NOTIFY_FAILURE_THRESHOLD`/`_WINDOW_MS`, `NOTIFY_WEEKLY_ENABLED`/`_CRON`. No migration (reuses `request_log`/`request_attempt`). Budget producers (`budget_alert`/`budget_block`) remain #16.
