Ship the self-host distribution (#22, spec §13/§14.11): `docker compose up` → a working instance on one port.

- **Production image** (`Dockerfile`, multi-stage `node:24-alpine`): builds the workspace, prunes dev deps, and keeps the monorepo layout so the SPA dist and bundled migrations resolve unchanged. Non-root `node` user, exec-form CMD (SIGTERM → Nest shutdown hooks: stream drain, writer flush, span flush), container healthcheck on `/api/health`.
- **Product compose** (`docker-compose.yml`, project `polyrouter-selfhost`, distinct from the dev-infra compose): app + postgres 16 + redis 7 with health-gated startup, `stop_grace_period: 45s`, and an optional `caronc/apprise` service behind the `apprise` profile (wired via the documented `APPRISE_API_URL` + port-bounded `NOTIFY_ALLOWED_ENDPOINTS` pair — the SSRF guard stays intact; the network subnet is pinned/overridable so the CIDR is deterministic). The app **publishes on loopback by default** — the unauthenticated first-sign-up-becomes-admin window and `/metrics` are never network-exposed until the operator claims the instance and opts in (`POLYROUTER_HOST`/`APP_URL`). Compose owns the container env (`BIND_ADDRESS=0.0.0.0`, service-network DB/Redis URLs); secrets interpolate from `.env` with `:?` guards; optional vars pass through as bare keys (unset stays unset).
- **Installer** (`install.sh`): checks Docker + Compose v2, fetches one source archive at a single ref (branch/tag/SHA via `POLYROUTER_REF`; the compose file comes from inside it), writes a mode-600 `.env` atomically (four 32-byte-hex secrets + DB password) that lives OUTSIDE the replaceable source tree and is **never rotated** on re-runs (rotating the encryption keys would orphan stored credentials), and boots the stack. Checkout mode uses the working tree and downloads nothing.
- **Self-contained SPA:** the Geist/Geist Mono fonts are vendored locally (variable woff2 + OFL license); the Google Fonts links are gone — the packaged dashboard loads with zero third-party fetches.
- **Fix (production-only, caught by the in-container smoke pass):** `configureSpa`'s fallback swallowed `GET /metrics` with the SPA shell — the `/metrics` namespace now passes through to Nest, with a prod-topology e2e guard.
- **Verification:** `scripts/selfhost-smoke.sh` — a repeatable in-container pass proving health/SPA/metrics/local fonts/loopback publish, exactly-one-admin bootstrap, a LIVE streamed completion draining to `[DONE]` across `docker stop` (inspected container exit state 0), metadata-only persistence (prompt sentinel absent from all request tables), restart with data intact, and the apprise profile booting with its allowlist pair.
- README gains the full Self-hosting section (exposure model, `.env` reference, rotation/`POSTGRES_PASSWORD`/single-replica caveats, upgrade/backup, the §16 subscription-ToS note).
