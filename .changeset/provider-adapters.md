Add the provider-call layer (spec §8, §3.2, §7.4; invariants 2/6/8/10) in `data-plane/src/providers/`: one adapter interface (`chat`/`chatStream`/`listModels`/`testConnection`) over OpenAI-compatible and Anthropic-compatible providers across the four kinds, consuming #5's `Normalized*` IR (no provider defines its own response shape). Outbound HTTP is SSRF-guarded at connect time using #4's `assertUrlSafe` + `createGuardedDispatcher` (not the auto-closing `guardedFetch`, which would deadlock streaming) with the dispatcher lifetime tied to the response body and redirects rejected; the loopback exception is gated to `kind:'local'` under `MODE=selfhosted` and the factory rejects local providers under `MODE=cloud`. Transport is JSON with per-protocol auth (`Authorization: Bearer` vs `x-api-key` + `anthropic-version` + a `max_tokens` default) and a persistent streaming `TextDecoder`; a first-byte timeout composes with the caller's `AbortSignal` (caller cancellation is breaker-neutral). A typed error taxonomy splits `shouldFallback` from `breakerImpact` so a retired model falls back without disabling a healthy provider. A generation-versioned Redis-shared circuit breaker (`closed→open→half-open→closed`) admits a single half-open probe, ignores stale-generation completions, bumps the generation on lease reclaim, degrades to a per-instance in-memory breaker when Redis is down (never fully open), and treats a truncated stream (no terminal stop reason) as a tripping failure; the transition math is a pure function pinned to a Lua implementation by a real-Redis parity + concurrency suite. Credentials are passed only in the auth header and never logged or surfaced in errors. Consumed by #7 (management/catalog sync) and #10 (proxy routing/fallback).
