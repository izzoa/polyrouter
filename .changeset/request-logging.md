Add request logging — an immutable-cost RequestLog for every routed request, written off the hot path (spec §5/§7.5/§7.7/§3.2.4; invariants 4, 8, 9).

- **Schema**: a new `request_log` table (migration `0003`). `agent_id`/`provider_id`/`model_id` are **denormalized ids, not foreign keys** — an append-only audit row must survive (and not fail to insert on) a concurrent provider/model/agent deletion and keep the historical id; only `owner_user_id` keeps its FK. Stores the routing decision (`decision_layer`, `routing_reason`, `tier_assigned`), **uncached** input/output tokens + separate cache tokens, snapshotted unit prices (`*_price_snapshot` + `price_version_id`), `usage_estimated`, `cost` (nullable = price unknown), `duration_ms`, and `status`. USD-only; no prompt/response bodies.
- **Cost immutability** (invariant 4): the writer snapshots unit prices via #8's `resolveForModel` at the request-completion timestamp and computes `cost` once; there is no update path, so a later catalog price change never moves a recorded cost. `computeCost` returns null when the price is unknown **or** a non-zero cache component lacks its rate (never an understated cost).
- **Usage** (invariant 9): prefers the provider's `usage`; when missing/partial it estimates (`chars/4` over request + response text **and tool-call name/args**, so a tool-only response isn't zero output) and flags `usage_estimated` — no billing tokenizer on the hot path. Streamed usage is captured by an additive `ProxyCore` seam (`runBuffered` exposes the response; `openStream` resolves a `StreamOutcome` that merges usage via `mergePartialUsage` and settles exactly once — including on a pre-`next()` client disconnect). `RouteDecision` gains `tierKey`.
- **Async, failure-isolated writer**: `record()` allocates the row id, estimates usage, and enqueues a metadata draft (no DB call). The `LogWriter` resolves prices + computes cost + batch-inserts **per principal** (owner forced from the principal) with `ON CONFLICT (id) DO NOTHING` (retry-idempotent) and **bounded retry + backoff** wrapping the whole resolve-price+insert attempt; a bounded queue drops with a logged/metered count. It flushes on interval/size and in `onApplicationShutdown` (after #10's stream drain). A logging failure never fails or blocks the request. (Redis buffering is a documented scale option; the in-memory writer is correct across instances.)
- **Persistence**: a `requestLogs` accessor — `insertMany(principal, rows)` (idempotent, owner-forced) + tenant-scoped `list`/`findById` (invariant 5).

Reviewed over three codex rounds (design), which hardened cache-rate cost, best-effort durability, the FK deletion race, idempotent retries, per-principal batching, tool-aware/cache-aware estimation, and the pre-iteration-disconnect outcome. Backed by pure cost/estimation + `ProxyCore` unit tests, recorder + writer unit tests, tenancy port tests (idempotent, isolated), and a real-Postgres e2e (a routed request writes one no-body log; **cost-immutability across a catalog price change**; a log-write failure doesn't fail the request; tenant isolation).
