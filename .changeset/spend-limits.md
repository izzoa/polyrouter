Add spend limits (#16, spec §5/§10; invariants 4/5/10/11) in `control-plane/src/budgets/`: owner-scoped `budget` CRUD at `/api/budgets` (a `global`/`agent`-scoped, `day`/`week`/`month`-windowed, `alert`/`block` budget with a USD `amount` and per-budget notification channels) over a new tenant-isolated `budget` table with DB check constraints, plus one migration.

Enforcement is an **asynchronously-metered postpaid soft cap** over **resetting UTC calendar windows**. The spend counter is a reconciled read-model in integer micro-dollars: a `budget-eval` scheduler (its own BullMQ queue, default-on, fail-open bootstrap) is the **sole writer** — each interval it recomputes every active budget's current-period spend from the immutable RequestLog ledgers (`request_log.cost` + cascade `request_attempt.cost`, row-level `round(cost×1e6)`) via a narrow cross-owner reader and monotonically sets the shared Redis counter (so it can't double-count or race), then stamps a reconciliation heartbeat. `block` budgets are enforced in the proxy request path (`ProxyService.completion`/`stream`, pre-`prepare`, streaming rejects pre-commit with a 402) via a bounded read on a dedicated fail-fast Redis connection off a capped in-memory budget cache; the read has a **named, configurable fail mode** (`BUDGET_FAIL_OPEN`, default open) that also covers a stale/absent heartbeat (a stopped scheduler is treated as enforcement-unavailable, never silently under-budget → fail-closed returns `503 budget_enforcement_unavailable`). `alert`/`block` events emit through #15a/#15b's non-blocking notification pipeline (once per budget per period, deduped in Redis; targeted to the budget's `notify_channel_ids` via a per-event fan-out intersection) — fire-and-forget, so a failing channel never blocks a request or enforcement. New env: `BUDGET_REDIS_TIMEOUT_MS`, `BUDGET_CACHE_TTL_MS`, `BUDGET_CACHE_MAX`, `BUDGET_FAIL_OPEN`, `BUDGET_SCHED_ENABLED`, `BUDGET_SCHED_CRON`, `BUDGET_STALE_MS`. Composite `(owner, created_at)` indexes on both ledgers support the reconcile scan. The reference spec §10 is updated to describe the resetting calendar window + single-writer reconciled counter the delta implements.
