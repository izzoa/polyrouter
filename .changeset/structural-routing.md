Add Layer 1 structural auto-routing — cheap, language-neutral structural pre-classification for `model=auto` (spec §7.1–§7.3, §7.6; invariants 1, 9).

- **Pure structural engine** (`data-plane/src/routing/structural/`): `extractStructuralFeatures` scores the last recent-message window **excluding the system block** (de-contamination by construction) and recursing into tool results; `classifyStructural` maps saturating, weighted sub-scores (effective input size — baseline-subtracted, code, tool count, tool-schema demand, multimodal, depth, max_tokens) to a `high` / `low` / `ambiguous` band. Language-neutral (no keyword matching), deterministic, `reason` carries only numbers.
- **Band → tier via `auto_high` / `auto_low` RoutingRules**: `RULE_MATCH_TYPES` gains two Layer-1 band targets that reuse #9's ownership-validated CRUD and are **inert in Layer 0**. `resolve.ts` exports `resolveTarget` + the rule comparator and adds the `'structural'` decision layer; a `tier:` target carries the tier's fallback chain, a `model:` target a single member.
- **Per-agent baseline, off the hot path**: an EWMA of effective input size per `(tenant, agent, system-fingerprint)` read from a bounded in-process LRU (synchronous — no network on the request path), with Redis as a bounded, best-effort shared backing over a dedicated `enableOfflineQueue:false` connection and a coalesced/throttled worker capped at a total transient budget. Baselines live in a per-`(tenant, agent)` hash with an atomic field cap + sliding TTL (hard cardinality bound); the hash field is a **server-keyed HMAC** of the framed system prompt — not correlatable and never recorded/logged (invariant 8).
- **Orchestration**: `StructuralRouter` runs only when `model=auto` fell through to the Layer-0 `default` decision (explicit models and `x-polyrouter-tier` headers still win), overriding it with `decision_layer='structural'` on a confident band with a resolvable target; **any failure — layer disabled, store unavailable, extraction error, missing/empty target, ambiguous band — degrades to the Layer-0 `default` and never fails or stalls the request** (invariant 1).
- **Gating + tunables** (`ROUTING_AUTO_LAYERS`, default `structural`, empty → pure Layer 0; `ROUTING_STRUCTURAL_HIGH_THRESHOLD` / `_LOW_THRESHOLD` / `_BASELINE_ALPHA` / `_WEIGHTS`) registered in the boot fail-fast schema, validated `0 ≤ LOW < HIGH ≤ 1`, `0 < ALPHA ≤ 1`, and weight shape. Default-on is behavior-preserving until `auto_high`/`auto_low` are configured.

Reviewed over three codex rounds (hot-path Redis removed, tenant-scoped/TTL'd/HMAC'd keys, concrete classifier + threshold validation, bounded background work, deterministic rule selection). Backed by pure feature/classifier unit tests, a real-Redis store spec (atomicity, field cap, TTL, cross-instance seed, bounded-outage), and a real-Postgres/stub-upstream e2e (complex→high, trivial→low, system-prompt de-contamination, learned-baseline escalation, header-wins, disabled→default). Native `reasoning_effort` / `response_format` signals and Layers 2/3 are explicitly deferred. No schema migration.
