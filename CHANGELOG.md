# Changelog

All notable, user-facing changes to polyrouter, newest first. Format follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/); versions follow the
frontend package's [semver](https://semver.org/) (the release version).

Per-package detail lives in `packages/*/CHANGELOG.md` (generated by changesets);
each release links its GitHub page, which carries the full narrative and
install/upgrade commands.

**Maintenance:** every user-facing change adds a changeset AND a one-line entry
under **[Unreleased]**. At release time, [Unreleased] is retitled to the new
version + date (matching the `chore(release)` commit), and a fresh [Unreleased]
heading is started.

## [0.8.0] — 2026-07-21

[Release](https://github.com/izzoa/polyrouter/releases/tag/v0.8.0) ·
[Compare](https://github.com/izzoa/polyrouter/compare/v0.7.0...v0.8.0)

### Added

- Per-provider **max-tokens field** for OpenAI-compatible providers: a
  `maxTokensSpelling` setting (`auto` | `max_completion_tokens` | `max_tokens`,
  default `auto`) chooses which wire field the output-token cap is sent under.
  `auto` is kind-derived — `local` providers emit `max_tokens`, every other kind
  emits `max_completion_tokens` (required by OpenAI o-series / reasoning models).
  Inbound still accepts either spelling; outbound emits exactly one. Set via the
  provider create/update API (a dashboard control is a follow-up).
- Optional Layer-2 semantic-embedder runtime (`SEMANTIC_MODEL_PATH` + a local
  model bundle): bounded local ONNX embedding with boot warmup, fail-fast on
  broken bundles, a content-derived model revision, and an ORT- and
  model-free baseline image (CI-asserted). `ROUTING_AUTO_LAYERS` becomes a
  validated token list and accepts an inert `semantic`; the auto-layers API
  reports `semanticAvailable`. Routing consumption arrives with the
  semantic-routing change.
- Layer-2 semantic routing: a Layer-1-ambiguous `auto` request is embedded and
  classified against bundled anchor centroids — a confident band routes via the
  existing `auto_high`/`auto_low` targets with `decision_layer='semantic'`, and
  anything ambiguous or faulted degrades to cascade/default unchanged. Four
  nullable telemetry columns (band/score/source/revision) with all-or-none DB
  checks, the ordered L1→L2 reason trail on default and cascade rows, a per-
  tenant `semantic` layer toggle (semantic⇒structural), and a
  `decision_layer=semantic` analytics filter. Opt-in, off unless an embedder is
  loaded; no prompt text or vectors are ever logged.
- Layer-2 **learning loop**: per-tenant learned centroids that track each tenant's
  own outcome-labeled traffic (opt-in, default OFF). Cascade outcomes weakly label
  the L2-ambiguous requests (quality-passed → `low`, quality-gate escalation →
  `high`); embeddings accumulate in bounded volatile memory and flush to Redis only
  as a ≥ `MIN_COHORT` sum — never a single raw embedding. A daily sweep folds fresh
  evidence into learned centroids under rails (min samples, capped EMA, spherical
  drift clamp toward bundled, cooldown, evidence-revision match), crash-atomically
  across Redis + Postgres (rotate → stage → CAS + scalars-only audit → promote).
  Learned supersedes bundled only under read-time gates; any failure or Redis fault
  falls back to bundled, never skip. One-action revert; a status view (counts /
  source / generation / last-applied). No vector ever reaches Postgres, a log, a
  metric, or an API response.
- Layer-2 semantic **dashboard**: the permanently-locked "L2 · Semantic" stub
  becomes a real driven toggle — honest available/unavailable copy naming
  `SEMANTIC_MODEL_PATH`, no dead control. A **learning card** (shown when L2 is
  effective) carries the opt-in learning toggle, fresh-sample / source / last-
  applied status, the numeric audit history, and a confirmed one-click revert —
  honest under degradation (a stale learned centroid shows `bundled` with the
  reason). Auto-performance gains the semantic slice (evaluated, routed-per-band,
  the four-way outcome split, bundled/learned source) with residual-cascade
  labeling so pre/post-enable cascade figures stay comparable, plus a
  `semantic_source` provenance chip in the request inspector. A
  **batteries-included `-semantic` image variant** ships beside the baseline:
  same Dockerfile (glibc base, exact-pinned ONNX runtime, reference MiniLM model
  pre-baked and checksum-pinned at build time, `SEMANTIC_MODEL_PATH` preset), a
  compose override with bring-your-own-model support, and a dual-arch release
  smoke test — the baseline image stays ORT- and model-free.

### Fixed

- Local / legacy OpenAI-compatible endpoints that accept only `max_tokens` no
  longer silently lose the caller's output-token cap — the proxy now sends the
  field each endpoint accepts (see the max-tokens field setting above). Existing
  `local` providers switch to `max_tokens` on upgrade; all others are unchanged.

### Upgrade notes

- **Migrations run on boot** (`0020` semantic telemetry columns on `request_log`;
  `0022` the `semantic_learning_event` table). No manual step — but back up
  before upgrading a production database, as always.
- **Layer-2 semantic routing is opt-in and off by default.** The baseline image
  is unchanged (no ONNX runtime, no model). To turn it on, either run the new
  **`ghcr.io/izzoa/polyrouter:0.8.0-semantic`** image (batteries-included: ORT +
  reference model pre-baked, `SEMANTIC_MODEL_PATH` preset) or install the
  `onnxruntime-node@1.27.0` peer and point `SEMANTIC_MODEL_PATH` at a model
  bundle. Add `semantic` to `ROUTING_AUTO_LAYERS` to expose the capability;
  tenants still opt in per-account. New knobs: `SEMANTIC_*` and
  `SEMANTIC_LEARNING_*` (all defaulted; learning is off until a tenant enables
  it). Nothing is fetched at runtime; no prompt text or embedding vector is ever
  logged or persisted.
- **`local` providers now emit `max_tokens`** (not `max_completion_tokens`) —
  a fix for endpoints that reject the newer spelling. Override per provider via
  the max-tokens field if your local gateway expects the other spelling.

## [0.7.0] — 2026-07-21

[Release](https://github.com/izzoa/polyrouter/releases/tag/v0.7.0) ·
[Compare](https://github.com/izzoa/polyrouter/compare/v0.6.0...v0.7.0)

### Added

- Per-provider upstream-timeout overrides (1s–1h, provider form → Advanced),
  resolved per fallback-chain attempt, with the instance defaults served by
  `GET /api/providers/timeout-defaults` for honest inherit display.

### Fixed

- Long-running research-class models (o3-deep-research and kin) survive
  end-to-end: the hidden undici 300s headers/body ceiling is gone (dispatcher
  timeouts now derive above polyrouter's typed bounds), and upstream bytes —
  SSE keepalive comments included — count as stream liveness, re-arming the
  stall watchdog and the breaker's half-open probe lease. True silence still
  fails typed at exactly the configured bound.
- The ChatGPT/Responses adapter's buffered path gained its own byte-re-armed
  idle guard (it previously had no typed post-headers bound).
- The body-capture card's buttons use the design system's real classes —
  "Purge all bodies" no longer renders as bare oversized text.

## [0.6.0] — 2026-07-20

[Release](https://github.com/izzoa/polyrouter/releases/tag/v0.6.0) ·
[Compare](https://github.com/izzoa/polyrouter/compare/v0.5.1...v0.6.0)

### Added

- **Pricing stays current by itself** — a daily automatic LiteLLM catalog
  refresh, on by default for self-hosted (`PRICING_REFRESH_SCHED_ENABLED=false`
  opts out), plus a Settings **Pricing catalog** panel with freshness, a
  "never refreshed" callout, and Refresh-now. New prices apply to new requests
  only; recorded costs never change.
- **Opt-in prompt & response body capture — off by default.** Three-way mode
  (Off / Errors & escalations only / All) behind an explicit consent confirm;
  per-agent overrides that are inert while the global mode is off; 256 KiB/
  direction caps with honest truncation; media stripped to size markers;
  encryption at rest always; 30-day retention (infinite only as an explicit
  choice) with purge-all, per-request delete, and keep-or-purge on disable —
  race-proof against in-flight writes. The inspector gains a lazily-fetched
  Payload section. Cloud instances never capture.
- **The inspector shows which header chose the route** — a dedicated `header`
  row for header-routed requests (`x-polyrouter-tier: heavy`, or a custom
  rule's header name only — configured rule values can be credentials and are
  never persisted).

### Changed

- **`x-polyrouter-tier` beats every routing mechanism except the `model`
  field** — the tier header is one coherent resolution phase (value remaps,
  then direct tier lookup) evaluated before rules on any other header; a
  matched-but-broken tier ask surfaces a clear error instead of silently
  rerouting.

### Fixed

- The Auto performance card refreshes on every Routing-page visit instead of
  freezing at its first fetch.

### Upgrade notes

- Migrations run on boot: `0016` (routing-header columns), `0017` (body-capture
  tables + per-agent override column). No backfill.
- Body capture requires `MODE=selfhosted` and an explicit enable in Settings;
  `BODY_CAPTURE_MAX_BYTES` (default 262144) tunes the per-direction cap.

## [0.4.0] — 2026-07-19

[Release](https://github.com/izzoa/polyrouter/releases/tag/v0.4.0) ·
[Compare](https://github.com/izzoa/polyrouter/compare/v0.3.0...v0.4.0)

### Added

- **`model: auto` honors declared complexity** — OpenAI `reasoning_effort`,
  Anthropic `thinking` budgets, and `output_config.effort` feed Layer-1;
  undeclared requests classify byte-identically to before.
- **Decision telemetry on every evaluated request** (`structural_band` /
  `structural_score` / `structural_band_source`), including ambiguous and
  unroutable fall-throughs.
- **The Auto-performance view** — band mix over time, the four-way cascade
  outcome split, unroutable diagnostics, and an estimated net-savings figure
  with visible coverage (never a fabricated $0).
- **Rail-bounded self-calibration (opt-in per tenant)** — conservative,
  audited, revertible narrowing of the ambiguous band from your own outcomes;
  escalations record WHY (`quality_gate` vs `cheap_error`).
- **Request error detail** — failed requests record the terminal provider
  error (kind, status, sanitized message, provider request id) into the
  inspector.
- **Native-family price fallback** with explicit estimate provenance.

### Changed

- A sharper cascade quality gate: unparseable structured output escalates;
  length-truncation grades 0.5; tool-calling turns are exempt.

### Upgrade notes

- Migrations `0011`–`0014` run on boot. Six new `CALIBRATION_*` env keys
  (defaults safe; opt-in per tenant). Threshold envs are boot-validated to at
  most 4 decimal places.

## [0.3.0] — 2026-07-18

[Release](https://github.com/izzoa/polyrouter/releases/tag/v0.3.0) ·
[Compare](https://github.com/izzoa/polyrouter/compare/v0.2.0...v0.3.0)

### Added

- **Subscription OAuth** — connect Claude Pro/Max and ChatGPT Plus/Pro with a
  guided wizard (PKCE exchange, encrypted token storage, rotation-safe
  auto-refresh, one-click Reauthorize); both presets verified live.
- **ChatGPT Responses protocol** (`openai_responses`) — streaming-only
  upstream served transparently to buffered clients; `store: false` on every
  call; reasoning items never persisted.
- **Provider-listed prices** captured at sync as display-only estimates with
  clear provenance (recorded cost stays on the bundled catalog).
- **Edit providers** from the dashboard with a write-only credential contract.
- OpenRouter app-attribution headers.

### Upgrade notes

- Migrations `0009`–`0010` run on boot. Rotating `PROVIDER_CREDENTIAL_KEY`
  invalidates stored credentials (OAuth providers ask to be reauthorized).
- Reusing flat-rate consumer subscriptions programmatically may violate
  provider ToS — pair them with a pay-per-token fallback (surfaced in the UI).

## [0.2.0] — 2026-07-17

[Release](https://github.com/izzoa/polyrouter/releases/tag/v0.2.0) ·
[Compare](https://github.com/izzoa/polyrouter/compare/v0.1.0...v0.2.0)

### Added

- **User administration** — atomic first-signup-wins bootstrap (registration
  then closes to invite-only), hash-stored single-use invites delivered by
  SMTP or copyable link, a Users admin page (promote/demote, disable/enable,
  delete, revoke invites, live registration toggle), and a sidebar account
  menu. Disabling a user cuts sessions AND their agent keys in one
  transaction; the last enabled admin is undeletable.
- **Hermes Agent harness** — paste-ready `~/.hermes/config.yaml` snippet on
  agent create/rotate.

### Upgrade notes

- Migration `0008` seeds existing instances to `invite_only` — public sign-up
  closes on upgrade until an admin reopens it.

## [0.1.0] — 2026-07-17

[Release](https://github.com/izzoa/polyrouter/releases/tag/v0.1.0)

First release: a self-hostable LLM router/gateway — one OpenAI- and
Anthropic-compatible endpoint with explicit-first routing and opt-in smart
layers (structural + cascade), commit-safe streaming fallbacks, golden-file-
tested cross-protocol translation, immutable per-request cost snapshots,
Redis-atomic budgets, async SMTP/Apprise notifications, a SolidJS dashboard
with a routing-decision inspector, encrypted credentials, HMAC agent keys,
SSRF-guarded egress, central tenant isolation, and single-container packaging
with Prometheus metrics + optional OpenTelemetry. AGPL-3.0-only.
