# Add Layer 3 cascade routing (cheap-first + escalation; reaches the L0+L1+L3 self-host target)

## Why

Layer 1 (#13) exits confident `auto` traffic to a tier, but leaves **ambiguous** requests on the Layer-0 `default`. Spec §7.2 Layer 3 is the escape hatch for exactly that uncertainty: **predicting difficulty is hard; detecting a bad answer is easy.** For an ambiguous, cheap-to-try request, route to the **cheap** model first, run a **cheap quality check**, and **escalate** to a stronger tier only when the cheap answer is bad (FrugalGPT-style). This reaches the honest self-host feature target — **L0 + L1 + L3** (§7.6) — with no embedding model and no per-request generative classifier (invariant 9).

The bar (spec §15, TODOS #14): a failing cheap answer escalates and the client still gets **one coherent response**; a **committed stream is never swapped** (invariant 3); the escalation is recorded and inspectable (`escalated=true`, `quality_signal`); disabling the layer changes nothing for explicit traffic.

## What Changes

- **Trigger on Layer 1's `ambiguous` verdict.** `StructuralRouter` is refactored to expose a discriminated evaluation (`route` | `ambiguous` | `skip`) instead of just `RouteDecision | null` (its `decide` behavior is unchanged for the `route` case). Cascade engages when an `auto` request evaluates `ambiguous`, `cascade` ∈ `ROUTING_AUTO_LAYERS`, and both `auto_low` (cheap) + `auto_high` (strong) band targets are configured — otherwise the Layer-0 `default` stands (degrade).
- **Pure cascade primitives** (`data-plane`): `evaluateQuality(response) → number` (a language-neutral, tokenizer-free quality **score** in `[0,1]` — empty output, malformed tool arguments (`inputParseError`), or an error stop reason drive it low) and `responseToStreamEvents(response) → NormalizedStreamEvent[]` (synthesize a stream from a buffered response so a passing cheap answer can be replayed to a streaming client via the existing `client.streamSerialize`). `DecisionLayer` gains `'cascade'`; `resolveTarget`/comparator are reused (a shared `resolveBandTarget` helper backs both #13 and cascade).
- **Cascade orchestration honoring the commit boundary** (`ProxyService` + a `CascadeRouter` policy object): the **cheap** tier is always run **buffered** (`runBufferedChain`, under a bounded cheap-response deadline so a hung upstream still escalates), even for a streaming client, so invariant 3 holds by construction; a client disconnect during the cheap attempt stops without calling the strong tier. Then:
  - `score ≥ ROUTING_CASCADE_QUALITY_THRESHOLD` (default `0.5`) and the cheap chain served → **deliver the cheap answer** (`escalated=false`); a streaming client gets a replay from a dedicated `replayBufferedStream` that **pre-materializes** the SSE frames from the buffered response, so a synthesis/serialize failure (before any byte) safely escalates instead of erroring a valid answer.
  - otherwise (bad answer, cheap chain failed, or cheap deadline) → **escalate**, walking `strong ++ default` (the strong tier's chain followed by the Layer-0 default bundle) so the reliable core still serves if the strong tier is down (invariant 1): non-streaming → buffered; streaming → a live `openStreamChain`.
  Only ever **one** tier's output reaches the client — **never a mid-stream swap**; a post-commit failure of the committed stream is #10's terminal error frame.
- **Recording — served summary + a per-call cost ledger** (new `request_attempt` table + migration; invariant 4): the `request_log` row stays the one-per-request **served summary** (unchanged from #11 — its `cost` = the served member) with `decision_layer='cascade'`, `escalated`, `quality_signal` (the binary score, or `null` on a fail-open quality error), `tier_assigned` = the served tier (correctly `default` when a default member serves after strong exhausts, via a per-attempt `tierKey`), served model/price snapshot. A new **`request_attempt`** table (FK → `request_log`, owner-scoped) records the **additional billable calls** — the superseded cheap attempt on an escalation — each at its own immutable snapshot price/usage/cost. **Total spend = `request_log.cost` + Σ `request_attempt.cost`** (a uniform formula: non-cascade and cascade-pass write no attempt rows, so it reduces to the request_log cost). The streamed passing-cheap `request_log` row takes its billable usage from the fully-buffered `cheap.response` (not the client-delivery accumulator, which a disconnect would truncate); only the delivery status comes from the replay outcome. `escalated`/`qualitySignal` thread through `RecordOutcome` → draft → `toRow`; `RequestRecorder.record` returns the request id so the cheap attempt is linked via `recordAttempt(requestLogId, …)`.
- **Gating + tunables**: `ROUTING_AUTO_LAYERS` gains the `cascade` token; a new `ROUTING_CASCADE_QUALITY_THRESHOLD` (`(0,1]`, default `0.5`, so a `0` score always escalates) and `ROUTING_CASCADE_CHEAP_TIMEOUT_MS` (default `30_000`) registered in the boot fail-fast schema. Cascade consumes Layer 1's ambiguity signal, so **enabling `cascade` implies `structural`** (the loader adds it to the effective set) — no silent no-op config.

## Impact

- Affected specs: **new capability `cascade-routing`**.
- Affected code: pure `evaluateQuality` + `responseToStreamEvents` and `'cascade'` `DecisionLayer` + `resolveBandTarget` in `data-plane`; `StructuralRouter` returns a discriminated `evaluate()` (with `decide()` kept as a #13 adapter); a `CascadeRouter` policy object + cascade orchestration in `ProxyService` (`Prepared` gains an optional cascade bundle; `completion`/`stream` branch on it, building the escalation chain as `strong ++ default` and running the cheap attempt under a deadline; a factored chain→attempts helper); `RecordOutcome`/`RequestLogDraft`/`toRow` gain `escalated` + `qualitySignal`; a new `request_attempt` table + owner-scoped persistence accessor + a `recordAttempt` path in the recorder/writer; `ROUTING_CASCADE_QUALITY_THRESHOLD` + `ROUTING_CASCADE_CHEAP_TIMEOUT_MS` + cascade-implies-structural in `routing.config`. Reuses #10 commit gate, #12 chain walkers + commit boundary + drain, #13 structural classification + band targets, #11 recording (request_log unchanged). **One migration** (the `request_attempt` table; the `escalated`/`quality_signal` columns already exist from #11). Size **L** (the cost ledger + streaming replay lift it from the original M).
- **Out of scope (declared)**: verifier / self-consistency / LLM-judge quality checks and low-self-reported-confidence signals (they need extra model calls — a cloud/opt-in extension of the pure structural checks, §7.6); refusal detection (inherently linguistic — deferred with the language-neutral bar in mind); Layer 2 semantic classification + the learning loop (cloud-tier, deferred); tuning cascade on non-`auto` or confident traffic (cascade is only for L1-ambiguous `auto`). Explicit/header/default and #13 structural routing are unchanged.
