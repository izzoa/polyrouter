# Add the inference proxy core (Layer 0 — explicit routing)

## Why

This is the shippable product (spec §7.2 Layer 0, §14 milestone 5): the public endpoints an external agent points at. Everything so far — translation (#5), provider adapters (#6), provider/model catalog (#7), pricing (#8), routing config (#9) — exists to serve one request path. This change wires them into the OpenAI- and Anthropic-compatible proxy with **explicit routing only** (invariant 1): name a model, a tier, `auto` (→ default tier for now), an `x-polyrouter-tier` header, or fall through to `default`. Streaming and non-streaming both work, with **slow-client backpressure from day one** (spec §3.2.5, invariant 12) and the **mid-stream commit boundary** (invariant 3) respected even though the fallback chain itself is #12.

The acceptance bar (spec §15): an agent configured with only `base_url = <router>/v1` + an API key + `model: auto` gets working completions, streaming and non-streaming; naming a model or forcing a tier via header works; and an OpenAI-shaped client reaches an Anthropic upstream (and vice-versa) for plain, multi-turn-tool, and streamed turns.

## What Changes

- **Public endpoints** (agent-key plane, `@UseGuards(AgentApiKeyGuard)` from #3, which stamps `last_used_at`): `POST /v1/chat/completions` (OpenAI wire), `POST /v1/messages` (Anthropic wire), `GET /v1/models`. The guard accepts **either** `Authorization: Bearer` (OpenAI SDK) **or** `x-api-key` (Anthropic SDK) so both are drop-in.
- **Route resolution** (ordered phases, spec §6.1/§7.2): `model` field (`auto`→default tier / provider-qualified or unambiguous bare model / tier key) → matching custom `header` RoutingRule (#9) → built-in `x-polyrouter-tier` header → `default`-match rule → the `default` tier. A resolved tier uses its **primary** (position-0) model; the resolver sorts rules itself and returns a typed `ambiguous_model` error rather than guessing when a bare id is on two providers. The fallback chain is #12. The resolver emits `decisionLayer` + `routingReason` for #11.
- **Any client protocol reaches any provider protocol** via #5: the client adapter does client-wire ↔ IR, the #6 provider adapter does IR ↔ provider-wire. The IR's `model` is retargeted to the resolved provider's external model id.
- **Streaming + backpressure**: SSE for `stream: true`, buffered JSON otherwise. A commit-gated coordinator waits for the first *successful* event (an upstream that throws **or yields an error event first** stays pre-commit); the async-generator pull chain plus `res.write`-drain (raced against client disconnect) gives end-to-end backpressure with no unbounded buffer and no hung drain. The **mid-stream commit rule** holds: pre-commit failure → clean protocol HTTP error; post-commit failure → a **sanitized** protocol-correct terminal error event, never a model swap (fallback is #12).
- **Graceful drain** (invariant 12): shutdown hooks + an in-flight-stream registry drain active completions on deploy and refuse new inference with a 503.
- **Protocol-shaped errors** via a `/v1` exception filter covering **every** failure (incl. the guard's 401): returned in the client's own envelope (OpenAI `{error:{…}}` / Anthropic `{type:"error",…}`) with an exhaustive status map, never leaking upstream body/credential. The **empty-tier / unresolved-target runtime error** promised by #9 is defined and enforced here.
- **`GET /v1/models`** lists the tenant's models plus tier keys plus the `auto` alias (OpenAI list shape).
- **Architecture**: the framework-agnostic engine — `resolveRoute` **and** a `ProxyCore` (translation orchestration + the commit-gated stream coordinator + a #10-local protocol terminal-error emitter, keeping #5 untouched) — lands in **`data-plane/src/{routing,proxy}/`**; only Express-specific pumping + guard/`@CurrentPrincipal` + persistence + credential decryption land in **`control-plane/src/proxy/`**. This respects the dependency direction (data-plane must not import control-plane) and makes the cloud extraction (§3.3) a lift-of-the-engine, rewriting only the pump.

## Impact

- Affected specs: **new capability `inference-proxy`**.
- Affected code: new `data-plane/src/routing/` (pure resolver) + `data-plane/src/proxy/` (`ProxyCore` + stream coordinator + terminal-error emitter); new `control-plane/src/proxy/` (controllers, `ProxyService`, exception filter, Express pump, drain registry); `control-plane/app.module` registers `ProxyModule` (`imports: [DatabaseModule, AuthModule]`). A small additive extension to #3's `AgentApiKeyGuard` (accept `x-api-key` alongside Bearer). Reuses #5 translate, #6 adapters, #9 accessors, #7 credential decryption. No schema migration.
- **Out of scope** (kept out deliberately): the fallback chain + circuit-breaker wiring (#12), the RequestLog write (#11), the `auto` smart pipeline L1/L3 (#13/#14) — `auto` is accepted and resolves to `default` now (spec §2). Tenant isolation (invariant 5) holds — all config is read through the owned persistence port for the guard's principal.
