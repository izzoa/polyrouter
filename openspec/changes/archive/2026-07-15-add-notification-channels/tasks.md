## 3. SSRF helpers (shared, reuse #4)

- [x] 3.1 `shared/security/network-host.ts`: `assertNetworkHostSafe(host, port, opts: { mode; allowedEndpoints? }): Promise<{ ip: string }>` — resolve the host (reuse #4's resolver), classify **every** resolved IP (`classifyIp`); **same policy shape as #4**: metadata/link-local/hard blocked in every mode (never allowlistable); **loopback** allowed **only in self-host** (`allowLoopback`), blocked in cloud; **soft private** blocked unless a **port-bounded** allowlist entry permits it, in **both** modes; throws `SsrfError`; returns a validated IP for connect-time pinning. Export from the server index.
- [x] 3.2 `shared/security/apprise.ts`: `assertAppriseTargetSafe(rawUrl, opts): Promise<void>` — parse the scheme; host-bearing set (http/https/ntfy/gotify/json/jsons/form/xml/matrixs/**mailto/mailtos**) → validate the URL host **and** host-override query params (`smtp`, `host`) via `assertNetworkHostSafe`; fixed-public-service set (discord/tgram/slack/pover/gchat/…) → allow; unknown scheme → reject (fail-closed). Export the maintained scheme sets. Export from the server index.

## 4. Event contract + notification service

- [x] 4.1 `notifications/notification.types.ts`: `EventType` union, a **structured secret-free** `NotificationEvent { type, scope:{ownerUserId, providerId?, agentId?, limitId?}, fields }` where `fields` is a typed non-secret per-type record (no free-form title/body, no URLs/tokens); pure `scopeKey(event): string` (canonical per type) + a per-type `windowMs` policy map; a `renderEvent(event): { title, body }` (fixed templates, in the worker).
- [x] 4.2 `notifications/notification.service.ts`: `NotificationService.emit(event): Promise<void>` — delegates to `NotifyQueue.enqueueFanout` (`queue.add('fanout', { event }, { deduplication: { id: hash(type,scopeKey), ttl: windowMs }, ...opts })`, opaque id, TTL-based dedup — no `:` in ids, survives job removal); **wrap in try/catch** so a Redis-down / deadline reject is logged (opaque) + swallowed, never blocking or thrown (invariant 11). No DB query on this path.

## 5. BullMQ delivery + adapters

- [x] 5.1 `notifications/notify.queue.ts`: producer `Queue('notify')` on `REDIS_CLIENT.duplicate({ maxRetriesPerRequest: 3, connectTimeout: 2000, commandTimeout: 2000 })` + `Worker('notify', processor, { connection: REDIS_CLIENT.duplicate({ maxRetriesPerRequest: null }), concurrency: 5 })`, both **eagerly connected** (base is `lazyConnect`). `enqueueFanout` wraps `queue.add` in a **`withDeadline(2000ms)`** → bounded `emit` even against a blackholed Redis (offline queue left enabled so a post-boot connect window doesn't drop emits; `commandTimeout` alone can't bound `waitUntilReady`). Attach an `'error'` listener to **both** `Queue` and `Worker` (an unhandled EventEmitter error on a down Redis crashes the process). Job opts: `attempts:4`, exponential backoff, `removeOnComplete:{age:3600}`, `removeOnFail:{age:86400}`. `OnApplicationShutdown` **deadline-bounds** `worker.close()`/`queue.close()` then force-disconnects (a worker blocked on a dead Redis mustn't hang shutdown). `whenReady()` awaits producer+worker readiness.
- [x] 5.2 Processor — `fanout`: load the owner's `enabled` channels subscribed to `event.type` (DB here, off the emit path) → enqueue a `deliver` job per channel `{channelId, ownerId, event}` with an **opaque** `jobId = hash(type,scopeKey,channelId,windowBucket)` (no `:`). `deliver`: load the channel (owner-scoped) → skip if missing/disabled → `decryptSecret` config → `renderEvent(event)` → dispatch by kind under a per-send timeout. A failure after all attempts throws a **sanitized code** (no secret) → logged (code + opaque job id only) + isolated.
- [x] 5.3 `notifications/delivery/smtp.adapter.ts`: `deliverSmtp(config, rendered, rt, timeoutMs)` — **connect-time** `assertNetworkHostSafe(host, port, rt)` → nodemailer transport **pinned to the validated IP** (`host: ip`, `tls:{ servername: host }`), `secure`/`requireTLS`/`ignoreTLS` from mode, `auth?`, connection/greeting/socket timeouts; `sendMail`; `transport.close()` in `finally`; catch raw error → throw `'smtp_send_failed'`. `notifications/delivery/apprise.adapter.ts`: `deliverApprise(config, rendered, rt, timeoutMs)` — **enforce the cloud egress gate here** (`mode==='cloud' && !appriseEgressConfirmed` → `apprise_egress_unconfirmed`, so an existing/backup-restored channel can't deliver in cloud without attestation); then `guardedFetch(<APPRISE_API_URL>/notify, { method:'POST', headers, body: JSON.stringify({urls,title,body}), signal: AbortSignal.timeout(timeoutMs) }, { context:{mode}, allowedEndpoints })`; `!response.ok` → `apprise_http_<class>`; unreachable → `apprise_unreachable`; unset URL → `apprise_not_configured`; consume/cancel the body on every path. Never log the raw error.
