# Design: add-pricing-catalog

## Context

Provider `/models` endpoints rarely return prices, so cost tracking needs a shipped, versioned catalog and an effective-dated lookup ("what did this model cost at request time"). §7.7 names an effective-dated `ModelPrice` table as the clean way to version it, and invariant 4 makes cost immutable via a request-time price snapshot (#11) — which needs this lookup as its source. The `model`/`provider` tables and the persistence-port pattern (incl. the non-owned facility `users`) exist from #2; this change adds the pricing catalog on top and computes no cost itself.

## Goals / Non-Goals

**Goals:** an effective-dated, append-only, **provider-namespaced** `model_prices` catalog (global reference data) with input/output + cache-read/write prices and capability flags; a bundled catalog seeded idempotently on boot covering the §8 providers + a curated free set; a pure resolver (`resolveModelPrice` + `deriveModelKey`) that is **cost-conservative** (unknown → null, never another provider's price) and reusable by #11; a session-authenticated, admin-gated, append-only refresh/override API.

**Non-Goals:** RequestLog / cost computation / snapshotting (#11); a *scheduled/background* pull job (an admin-triggered LiteLLM pull lands here; a periodic BullMQ job is deferred); per-tenant/per-org catalogs; writing prices/capabilities back onto `Model` rows (consumers resolve via the catalog); any routing/proxy change (#9/#10).

## Decisions

1. **The catalog is provider-NAMESPACED via ONE shared canonicalizer, or it silently applies the wrong price (Round-1 High-1, Round-2 High-1).** Direct OpenAI, a reseller, a subscription plan, and a custom gateway can all expose `gpt-4o` at different economics, so keying by bare `external_model_id` is a cost hazard. `model_key` is `"<family>:<canonical_model_id>"`, and **the catalog side (from LiteLLM) and the lookup side (from a tenant provider) MUST produce byte-identical keys** — a single `canonicalModelKey(family, modelId)` helper is used by both, or major families (Gemini, OpenRouter) resolve unknown. Its rules: `family` is a **LiteLLM `litellm_provider` value** (`openai`, `anthropic`, `gemini`, `deepseek`, `mistral`, `groq`, `openrouter`, …); `modelId` is lower-cased and has exactly one leading `"<family>/"` stripped (LiteLLM keys models `gemini/gemini-1.5-pro`, `openrouter/…` while a provider `/models` returns the bare id). `deriveModelKey(providerBaseUrl, externalModelId) → string | null` maps a provider *host* to that exact family via `PROVIDER_FAMILY_HOSTS` (`api.openai.com→openai`, `api.anthropic.com→anthropic`, `generativelanguage.googleapis.com→gemini`, `api.deepseek.com→deepseek`, `openrouter.ai→openrouter`, …) and then `canonicalModelKey`; an **unknown host returns null** → no catalog price applied. Real Gemini/OpenRouter fixtures pin the round-trip (catalog key == derived key).

2. **`model_prices` is a GLOBAL, effective-dated, append-only table with price-integrity constraints.** Non-tenant reference data (no owner columns), exposed via a non-owned `pricing` port accessor (mirroring `users`). Columns: `id`, `model_key` (namespaced text), `input_price_per_1m`, `output_price_per_1m` (double, NOT NULL, **CHECK ≥ 0**), `cache_read_price_per_1m`/`cache_write_price_per_1m` (nullable, **CHECK ≥ 0**), `context_window` (nullable), `supports_tools`/`vision`/`reasoning` (bool), `is_free` (bool), `source` (`bundled`|`refresh`|`manual`), `valid_from` (timestamptz NOT NULL), `created_at`. **Prices are USD per 1M tokens** (a normative single-currency invariant — no currency column, documented). A **CHECK enforces `is_free ⇒ input=0 AND output=0`**. Unique `(model_key, valid_from)` (the only index needed — the unique B-tree serves the DESC lookup, Round-1 Low-10). The service rejects non-finite (NaN/Infinity) rates and requires input+output together.

3. **One locked, monotonic, manual-respecting apply path for seed + refresh + override (Round-1 High-2, Round-2 High-2).** *Every* insertion path — boot seed, refresh, and manual override — goes through a single `applyVersions(entries, validFrom, source)` serialized by the persistence facility's `withAdvisoryLock` + a transaction, re-reading each key's `latest` inside the lock. Rules per key: **strict monotonicity** (skip a `valid_from ≤ latest.valid_from`, so nothing backdates history) and **manual protection** (a `bundled`/`refresh`/`litellm` entry is skipped when `latest.source === 'manual'`, so an operator override is never superseded; only a new `manual` override supersedes a `manual`). Timestamps are trusted: **refresh and override use the server-observed instant inside the lock** (not a client value); the **boot seed uses `BUNDLED_CATALOG_VERSION`** (a fixed date — a version bump moves forward monotonically; an older/rolled-back bundle is skipped). So `seedBundled` is not an unconditional `ON CONFLICT DO NOTHING` — it obeys the same latest/manual/monotonic checks. The *real* cost-immutability guarantee remains #11's request-time snapshot; this catalog just never lets a past lookup change.

4. **The resolver returns a full price snapshot and a distinct "unknown price" signal — NOT usage_estimated (Round-1 High-3).** `resolveModelPrice(input, catalogRow) → PriceSnapshot | null` is pure (no DB, no clock): precedence (a) the `Model` row's explicit user-entered prices (`source: model`); (b) else `local` provider → free (`0/0`, `source: local`); (c) else `catalogRow` (which the caller obtained via `priceAt`); (d) else **null**. A **null means "price unknown"** — a signal distinct from §7.7's `usage_estimated` (which is missing *token usage*); #11 records a missing-price case explicitly and never fabricates a cost. `PriceSnapshot` carries `{ priceVersionId, modelKey, inputPricePer1m, outputPricePer1m, cacheReadPricePer1m, cacheWritePricePer1m, isFree, source, validFrom }` so #11 can snapshot the exact rates + provenance + version; **cache rates stay nullable**, and #11 (not this layer) decides a `partial` cost when a request has cache tokens but the catalog lacks a cache rate.

5. **LiteLLM is the pricing source — for both the bundled snapshot and live refresh (§7.7).** The canonical data is LiteLLM's `model_prices_and_context_window.json` (a `{ "<model>": { input_cost_per_token, output_cost_per_token, cache_read_input_token_cost, cache_creation_input_token_cost, max_input_tokens, litellm_provider, mode, supports_function_calling, supports_vision, supports_reasoning, … } }` map). A pure `parseLiteLlmCatalog(json) → BundledPrice[]` maps each `mode: chat`/`completion` entry to a catalog row: `model_key = canonicalModelKey(litellm_provider, name)` (Decision 1's shared helper — strips a leading `"<provider>/"` so `gemini/gemini-1.5-pro` and a provider's bare `gemini-1.5-pro` produce the same key), prices `= cost_per_token × 1e6` (LiteLLM is per-token; we store per-1M USD), cache rates likewise, `context_window = max_input_tokens`, capability flags from `supports_*`, `is_free` when the input/output cost is 0. Non-`chat` modes (embedding/image/rerank), the `sample_spec` placeholder, and malformed entries are skipped. The parser is pure and unit-tested against real LiteLLM fixtures (incl. Gemini/OpenRouter prefixed keys).

6. **The bundled catalog is a committed LiteLLM snapshot; seed is idempotent + race-safe.** `pricing/bundled-catalog.ts` exports `BUNDLED_CATALOG_VERSION` (a UTC `Date`) and `BUNDLED_PRICES` — a **committed snapshot** produced by `parseLiteLlmCatalog` over a vendored LiteLLM JSON (a curated subset covering the §8 providers + free set, kept small and reviewable). `PricingBootstrap` (`OnApplicationBootstrap`, after migrations, before serving) calls `pricing.seedBundled(BUNDLED_PRICES, BUNDLED_CATALOG_VERSION)` — a bulk `INSERT … ON CONFLICT (model_key, valid_from) DO NOTHING`; concurrent boots converge on the conflict, and a content change MUST bump `BUNDLED_CATALOG_VERSION`.

7. **The port `pricing` accessor is the whole DB surface; it has no update/delete of price rows.** `priceAt(modelKey, at)`, `latest(modelKey)`, `listLatest()` (`DISTINCT ON (model_key) … ORDER BY model_key, valid_from DESC`, filtered to `valid_from ≤ now` so a future-dated row isn't shown as "current"), and `insertVersion(entry)` (append; rejects a non-monotonic `valid_from` as a backstop). There is **no bulk `seedBundled` and no update/delete** — the service's locked `applyVersions` (Decision 3) is the single write path (seed + refresh + override all read `latest` and call `insertVersion` inside the advisory lock), so the manual-protection and monotonicity rules can't be bypassed. History is immutable by construction; the unique `(model_key, valid_from)` is the final backstop.

8. **Live LiteLLM refresh is fetched behind #4's SSRF guard, with hard fetch limits (Round-1 High-6, per user request).** An admin refresh may pull the live LiteLLM JSON from `PRICING_REFRESH_URL` (config; default LiteLLM's raw GitHub URL). `fetchLiteLlmCatalog(url, { mode, timeoutMs, maxBytes, fetchImpl? })` uses #4's exported `assertUrlSafe` + `createGuardedDispatcher` + undici's `fetch` (NOT the auto-closing `guardedFetch`, whose graceful `dispatcher.close()` can hang on a ~MB body before it drains — the same lifecycle issue #6 hit). Hard limits: connect-time IP validation (private/loopback/link-local/metadata **rejected** — `providerKind` is unset so there is *no* loopback exception even in self-host); `redirect: 'manual'` with **any 3xx rejected** (a raw pricing JSON never legitimately redirects); an `AbortSignal` **timeout**; and a **max-body-size cap enforced while streaming** (abort once bytes exceed the cap — never buffer an unbounded body). It does **not** require a JSON content-type (GitHub raw serves `text/plain`); it validates by bounded JSON parse. On any failure (SSRF, size, non-2xx, timeout, parse) the response body is **cancelled/destroyed and the dispatcher closed** before throwing — no leak, no partial apply. The parsed catalog is applied through the locked `applyVersions` path (Decision 3). The `fetchImpl` seam lets the service unit/e2e inject a fake catalog for the success path, while the real guard is exercised by dedicated SSRF/size tests (a real successful pull can't hit loopback under the no-exception policy).

9. **Management API: session reads, admin-AND-self-host mutations (Round-1 High-5).** `api/pricing`: `GET /` (current catalog) and `GET /:modelKey?at=<ts>` (effective price) need only a session and expose **no tenant/credential data**. Mutations require the caller to be an **admin** *and* `MODE=selfhosted` (a cloud tenant-admin must not rewrite prices for every tenant → cloud disables these routes; admin is checked via a new `IdentityPort.isAdmin(userId)` on the `user.role` column, not the arbitrary `findAdminUserId()`): `POST /:modelKey/override` (append a manual version), `POST /refresh` (re-apply the bundled catalog, apply a request-body catalog, or — when `source: litellm` — pull the guarded LiteLLM URL per Decision 8).

## Risks / Trade-offs

- [Wrong-provider price] → the catalog namespace is LiteLLM's authoritative `litellm_provider`; resolution maps a tenant model to a family via the host allow-map with an unknown-host → null; no bare-id matching.
- [SSRF via the refresh URL] → the LiteLLM pull goes through #4's `assertUrlSafe` + guarded dispatcher (connect-time IP validation, no loopback exception, bounded redirects) with a timeout + max-body-size cap and defensive parse; it is admin-AND-self-host-gated; tested against a private/metadata URL, an oversized body, and non-JSON.
- [Historical resolution mutated by backdating] → monotonic `valid_from` (server clock for overrides; version-monotonic for refresh), refresh never clobbers a manual override, refresh serialized by advisory lock.
- [Unknown price hidden as a cost] → resolver returns null (distinct signal); #11 records missing-price and never guesses; cache rates carried nullable for #11 to mark `partial`.
- [Catalog capabilities vs Model rows] → this change does NOT write to `Model` rows; consumers (#7 listing, #9 routing, #11 cost) resolve capabilities/prices via the catalog. The proposal claim of "enriched Model rows" is dropped.
- [Cloud admin authority] → mutations gated on self-host + admin; cloud disables them (deploy-managed).
- [Price integrity] → CHECK (≥ 0), `is_free ⇒ 0`, finite-rate + both-prices validation, USD-only invariant.
- [Same-version content drift] → bumping `BUNDLED_CATALOG_VERSION` on any content change is a documented release discipline.

## Migration Plan

Additive: one new `model_prices` table (generated Drizzle migration, runs on boot before serving) with its CHECK constraints; one non-owned `pricing` accessor + an `isAdmin` method on the identity port (interface in shared, impl in the control-plane port); a new `PricingModule` (bootstrap/service/controller); pure `resolveModelPrice`/`deriveModelKey`/`parseLiteLlmCatalog` helpers in shared; a `pricing` config fragment (`PRICING_REFRESH_URL` default LiteLLM's raw URL, plus fetch timeout/max-body-size); and `undici` as a control-plane dependency (the guarded LiteLLM fetch uses its version-matched `fetch`). No change to existing tables, to #4/#6/#7, or to the tenant-scoped repos. Rollback = drop the module/accessor and the additive table.

## Open Questions

None blocking. A scheduled SSRF-guarded live pull, per-provider price divergence beyond the host allow-map, and a content-digest on the bundle version are deliberate later refinements — none needed for the immutable-cost foundation this change lays.
