# Design: add-protocol-translation

## Context

The proxy (#10) and adapters (#6) don't exist yet, but both depend on a normalization module ΓÇõ spec ┬º6.3 names it the hardest part of the app and CLAUDE.md invariant 2 requires it be its own module behind a `Normalized*` IR with golden-file contract tests. This change lands that module in `data-plane/src/proxy/translate/` (both future consumers are data-plane). It is pure: no HTTP, no DB, no config ΓÇõ just types + transform functions + a fixture-backed suite.

## Goals / Non-Goals

**Goals:** one `Normalized*` IR covering requests, responses, and streams for OpenAI Chat Completions and Anthropic Messages; pure per-protocol in/out adapters; correct handling of the ┬º6.3 divergences (system prompt, tool calls incl. multi-turn/parallel **and `tool_choice`/parallel controls**, streaming events **incl. the usage lifecycle**, stop reasons **incl. stop-sequence/refusal/continuation**, multimodal **incl. image detail**, usage **with correct cache-token accounting**); a per-adapter quirk mechanism; a golden-file suite proving **canonical round-trip equivalence** and cross-translation.

**Non-Goals:** the proxy/HTTP/routing (#10), fallbacks + mid-stream commit (#12), provider adapters + outbound calls (#6), live capture, protocols beyond the two, multiple choices (`n > 1` ΓÇõ single-choice contract, below).

## Decisions

1. **The IR is content-blocks-everywhere with a top-level system field.** `NormalizedMessage { role: 'user'|'assistant'|'tool', content: Block[] }`; `Block` is a discriminated union: `text`, `image`, `tool_use`, `tool_result`. `system` is a top-level `NormalizedRequest.system?: Block[]` (Anthropic-native; OpenAI's `role:system` message maps in/out of it). Tool input is a **parsed object** (not OpenAI's stringified `arguments`) ΓÇõ the thing that makes multi-turn tool round-trips correct. Generation params carried canonically: `maxOutputTokens` (ΓÓ╖ OpenAI `max_completion_tokens`/legacy `max_tokens`, Anthropic `max_tokens`), `temperature`, `topP`, `stopSequences`.

2. **`tool_choice` and parallel-call control are first-class IR fields (High-4).** Dropping them can turn a *forced* tool call into free text or permit calls the client forbade. `NormalizedToolChoice = 'auto' | 'none' | 'required' | { toolName: string }` on `NormalizedRequest.toolChoice?`, and `NormalizedRequest.allowParallelTools?: boolean` (default true). OpenAI: `tool_choice` (`none`/`auto`/`required`/`{type:'function',function:{name}}`) + `parallel_tool_calls`. Anthropic: `tool_choice.type` (`auto`/`any`/`tool`/`none`, `any`ΓÓ╖`required`, `tool`ΓÓ╖`{toolName}`) + `disable_parallel_tool_use = !allowParallelTools`. Every mode maps both ways; unsupported combinations are documented, not silently dropped.

3. **Tool results are one-per `role:'tool'` message; trailing user text is preserved (High-5).** OpenAI wants exactly one `role:'tool'` message per `tool_call_id`; Anthropic wants all `tool_result` blocks **at the start of a single `user` message**, optionally **followed by ordinary text**. The IR models each tool result as its own `role:'tool'` message carrying **exactly one** `tool_result` block (`{ toolUseId, content: Block[], isError? }`) ΓÇõ clean 1:1 with OpenAI. Anthropic `requestIn`: a `user` message of `[tool_resultΓÇª][text/imageΓÇª]` splits into N ordered `role:'tool'` messages **plus** (if trailing non-tool blocks exist) one following `role:'user'` message carrying them. Anthropic `requestOut`: a maximal run of consecutive `role:'tool'` messages becomes one `user` message with the `tool_result` blocks first, and if a `role:'user'` message immediately follows the run its blocks are appended as trailing content of that same user message. This round-trips (Anthropic disallows consecutive same-role messages, so at most one user message follows a tool run). Parallel `tool_use` blocks in an assistant turn keep their order.

4. **Usage uses uncached components, not field-for-field copies (Critical-1).** The two providers count input differently: **Anthropic `input_tokens` *excludes* cache tokens** (total input = `input_tokens` + `cache_read_input_tokens` + `cache_creation_input_tokens`), while **OpenAI `prompt_tokens` *includes* cached tokens** (`cached_tokens` is a subset). A field-for-field map corrupts usage and downstream cost (invariant 4, ┬º7.7). The IR therefore stores **components**: `NormalizedUsage { inputTokens /* uncached, fresh */, outputTokens, cacheReadTokens?, cacheWriteTokens? }`, with the documented identity `totalInput = inputTokens + (cacheReadTokens ?? 0) + (cacheWriteTokens ?? 0)`.
   - **Anthropic in:** `inputTokens = input_tokens`; `cacheReadTokens = cache_read_input_tokens`; `cacheWriteTokens = cache_creation_input_tokens`; `outputTokens = output_tokens`. **out:** reverse.
   - **OpenAI in:** `cacheReadTokens = prompt_tokens_details.cached_tokens`; `inputTokens = prompt_tokens ΓêÈ cacheReadTokens`; `cacheWriteTokens = undefined` (OpenAI auto-caches, no separate write billing); `outputTokens = completion_tokens`. **out:** `prompt_tokens = inputTokens + (cacheReadTokens ?? 0) + (cacheWriteTokens ?? 0)`; `prompt_tokens_details.cached_tokens = cacheReadTokens`; `completion_tokens = outputTokens`; `total_tokens = prompt_tokens + completion_tokens` (a cross-translated Anthropic cache-write folds into OpenAI `prompt_tokens`, which OpenAI bills at input rate ΓÇõ correct, since OpenAI has no cache-write price).
   - Missing usage stays `undefined` (proxy flags `usage_estimated`, #11) ΓÇõ never a silent zero. Numeric golden cases: cache hit, cache write, mixed, none, and a partial/impossible combination.

5. **Canonical stop reasons with raw passthrough and stop-sequence (Medium-7).** `NormalizedStopReason = 'stop' | 'length' | 'tool_use' | 'content_filter' | 'pause' | 'error' | 'other'`; `NormalizedResponse.rawStopReason?: string` always preserves the provider's original value, and `NormalizedResponse.stopSequence?: string` preserves Anthropic's matched sequence. Mapping ΓÇõ Anthropic: `end_turn`ΓÁÈ`stop`, `max_tokens`ΓÁÈ`length`, `tool_use`ΓÁÈ`tool_use`, `stop_sequence`ΓÁÈ`stop` (+`stopSequence`), `refusal`ΓÁÈ`content_filter`, `pause_turn`ΓÁÈ`pause` (continuation-required, kept distinct), unknownΓÁÈ`other`. OpenAI: `stop`ΓÁÈ`stop`, `length`ΓÁÈ`length`, `tool_calls`ΓÁÈ`tool_use`, `content_filter`ΓÁÈ`content_filter`, unknownΓÁÈ`other`. Reverse maps to each protocol's nearest native value (`tool_use`ΓÁÈ`tool_calls`/`tool_use`; `content_filter`ΓÁÈ`content_filter`/`refusal`; `pause` has no OpenAI equivalent ΓÁÈ `stop`); when the target protocol equals the source, `rawStopReason` is preferred for exact fidelity.

6. **Multimodal keeps encoding *and* detail; no fetching (Medium-8).** `image` block is `{ data, mediaType, detail? } | { url, detail? }` with `detail?: 'auto'|'low'|'high'`. OpenAI `image_url` (a `data:` URL or http(s) URL, with `detail`) and Anthropic `source` (`base64` or `url`) both normalize in; each adapter emits its native shape out. **`detail` survives an OpenAI same-protocol round-trip and any path that keeps the IR instance, but is *intentionally dropped* the moment content crosses an Anthropic wire boundary** ΓÇõ Anthropic's image source has no `detail` field, so an OpenAIΓÁÈAnthropic-wireΓÁÈIRΓÁÈOpenAI path loses it by construction (Round-2 High-3). This is documented as a dropped field in the affected golden, not asserted as preserved. A remote `http(s)` URL is preserved as `{ url }`, **never fetched** ΓÇõ fetching a user image URL is the proxy's SSRF-guarded concern (#6/#10), not the translator's.

7. **Streaming models the usage lifecycle and finalized blocks explicitly (High-3, Round-2 High-1).** Events: `message_start` (id, model, role, **and any partial `NormalizedUsage` the provider sends up front**), `text_delta` (index, text), `tool_use_start` (index, id, name), `tool_use_delta` (index, partialJson), `block_stop` (index, **`finalizedToolUse?`** ΓÇõ the parsed-or-`inputParseError` `ToolUseBlock` for a tool block, absent for text), `message_delta` (`stopReason?`, **partial/cumulative `NormalizedUsage`** ΓÇõ *not* output-only), `message_stop`, `error`. Provider realities: **Anthropic** sends input/cache usage in `message_start` and cumulative output tokens in `message_delta`; **OpenAI** (nominal, with `stream_options.include_usage`) sends the **complete** usage (input + cache + output) in a **final chunk whose `choices` is empty**, after the finish-reason chunk ΓÇõ so on the OpenAIΓÁÈIR path the input/cache components arrive at the *end*, on `message_delta`. Both `message_start` and `message_delta` therefore carry a *partial* usage, and the merge is **per-component** (`inputTokens`/`cacheReadTokens`/`cacheWriteTokens`/`outputTokens` each filled by whichever event supplies it; a later non-`undefined` component wins). An interrupted stream with no terminal usage leaves the missing components `undefined` (proxy estimates, #11). The serializer for an OpenAI target emits the finish-reason chunk then the empty-`choices` usage chunk. **Block-index allocation:** Anthropic uses one content-block index namespace over text *and* tool_use; OpenAI splits text (`delta.content`) from `delta.tool_calls[].index` (its own 0-based namespace). The OpenAI parser allocates canonical block indices in emission order ΓÇõ leading assistant text is block 0, then OpenAI tool_call `index` *i* maps to canonical block `(textBlockCount + i)` ΓÇõ so mixed text+tool streams get a single unambiguous index space. Tool-call JSON arrives as string fragments (both protocols), accumulated per canonical block index. The parser buffers partial SSE lines across chunk boundaries.

8. **Malformed tool-argument JSON is data, not an exception (High-6).** OpenAI documents that generated `arguments` may be invalid JSON. Translation **never throws** on model output: a `tool_use` block is `{ id, name, input: object }` on success or `{ id, name, inputRaw: string, inputParseError: true }` on parse failure (discriminated). Streaming accumulates `partialJson` per index and finalizes only at `block_stop`, which carries the finalized block as `finalizedToolUse` (per Decision 7); a failed parse there yields the `inputRaw`/`inputParseError` variant. Serializing that variant back to a protocol re-emits the raw string as the tool arguments. Golden cases cover malformed JSON both non-streaming and streamed.

9. **Single-choice contract (`n = 1`) (High-6 tail).** This layer normalizes a single assistant choice. Streaming tool-JSON assembly keys on the canonical content-block index within choice 0. An OpenAI request carrying `n > 1` is out of scope: the translator does not fan out multiple choices; #10's proxy decides policy (reject/clamp). Documented so the assembly keys and the response shape stay unambiguous.

13. **Target-required fields have a pure, deterministic serialization policy (Round-2 High-2).** Some fields one protocol requires have no source in the other, and the module is pure/deterministic (no `Date.now()`). (a) **`created`** ΓÇõ OpenAI responses require it; Anthropic responses have none. The IR carries `created?` as **optional**; OpenAI `responseOut` takes an optional pure serialization context `{ created?: number }` (the proxy supplies the request-time value it already has) and, absent both IR `created` and context, emits `created: 0` as a documented placeholder the proxy overrides ΓÇõ never a clock read. (b) **`max_tokens`** ΓÇõ Anthropic requests require it; OpenAI's is optional. The Anthropic adapter resolves it from IR `maxOutputTokens` or an adapter `defaultMaxOutputTokens` option; with neither it returns a **structured serialization error** (a precondition on *our* outbound request, not untrusted model output ΓÇõ so surfacing it is correct), and the proxy (#10), which knows the model's limit, always supplies one. Goldens: an OpenAI request with no token limit ΓÁÈ IR `maxOutputTokens` undefined ΓÁÈ Anthropic-out uses the default; an Anthropic response ΓÁÈ OpenAI-out with a supplied `created`.

10. **Provider quirks are a per-adapter option object, not sprinkled conditionals.** Each adapter takes an optional `quirks` for **genuine deviations from the nominal protocol** ΓÇõ e.g. a provider that omits `usage` entirely, or returns tool arguments already parsed. (OpenAI's usage-in-final-empty-chunk is *nominal*, handled in the core stream logic per Decision 7 ΓÇõ **not** a quirk.) The core transform stays clean; ships with an empty default; #6 populates per provider.

11. **Round-trip is canonical equivalence, not byte-identity (High-2).** A provider request/response carries fields the IR intentionally doesn't model (OpenAI `system_fingerprint`, `logprobs`; equivalent-but-different encodings like string-vs-block `content`, string-vs-array `system`, `max_tokens` vs `max_completion_tokens`). Byte-exact `Out(In(x)) ΓÊí x` is therefore impossible and dishonest to assert. The contract is `canon(Out(In(x))) ΓÊí canon(x)`, where `canon` coerces content/system to block-array form and normalizes the equivalent parameter encodings. The IR **does** carry response envelope identity ΓÇõ `id`, `model`, `created` ΓÇõ so responses keep identity across translation; fields with no IR home are documented as intentionally dropped and excluded from `canon`. The golden suite documents, per fixture, which fields are canonicalized vs. dropped.

12. **The mid-stream commit policy is stated, not enforced here.** ┬º6.3's rule (don't commit to the client stream until the first upstream token; never silently swap mid-stream) is a proxy/fallback concern (#12). This module provides the streaming parse/serialize primitives that make it implementable (the proxy inspects the first `NormalizedStreamEvent` before committing) and documents the boundary, but enforces no commit/fallback logic.

## Risks / Trade-offs

- [Cache-token accounting is the cost-correctness crux] ΓÁÈ components (uncached/read/write) with documented per-provider formulas and numeric goldens (hit/write/mixed/none/partial), not field-for-field copies.
- [Multi-turn/parallel tool mapping + trailing text is the bug magnet] ΓÁÈ one-result-per-tool-message + explicit group/split with trailing-user-text handling, isolated in the Anthropic adapter, with dedicated bidirectional goldens (two parallel results, one error result, trailing text).
- [Round-trip honesty] ΓÁÈ canonical equivalence with an explicit canonicalizer and a per-fixture record of canonicalized/dropped fields, rather than a false byte-exact claim.
- [Streaming usage placement differs per provider] ΓÁÈ explicit lifecycle (start-usage Γèò delta-usage), OpenAI empty-`choices` chunk handled in core, interrupted-stream and cache-in-stream goldens.
- [Malformed model JSON] ΓÁÈ represented as `inputRaw`/`inputParseError` data; translation never throws; streamed and non-streamed goldens.
- [Remote image URLs] ΓÁÈ carried as `{ url }`, never fetched (SSRF is #6/#10); the translator makes no network call.

## Migration Plan

Pure addition (new module + fixtures + tests). Rollback = remove the module.

## Open Questions

None blocking. Fixtures are hand-authored to the documented wire formats (no live keys); provenance/version is recorded in the suite README and refreshing from sanitized live captures is a maintenance follow-up ΓÇõ the canonical round-trip/cross-translation contract holds regardless of source. (Whether the IR later moves to `@polyrouter/shared` if a control-plane consumer appears is a deferred call; today both consumers are data-plane.)
