# Design: add-provider-adapters

## Context

The proxy (#10) resolves a route to a provider and must call it over that provider's wire protocol, guarded against SSRF, with a shared circuit breaker so a failing provider is skipped fast. This change lands that provider-call layer in `data-plane/src/providers/`, consuming #5's `Normalized*` IR (translation) and #4's SSRF primitives. It stores nothing and exposes no endpoints — #7 (CRUD/encryption) and #10 (proxy/routing/fallback) compose it.

## Goals / Non-Goals

**Goals:** one adapter interface (`chat`/`chatStream`/`listModels`/`testConnection`) over OpenAI-compatible and Anthropic-compatible providers spanning the four kinds; all outbound HTTP SSRF-guarded at connect time with correct streaming lifecycle; a typed error taxonomy with **separate fallback and breaker-impact classifiers**; a Redis-shared circuit breaker (`closed→open→half-open→closed`) with **admission tokens / generation epochs**, single-probe half-open, and per-instance degrade; credentials never logged.

**Non-Goals:** provider CRUD/persistence/encryption-at-rest (#7), pricing/catalog rows and `is_free` marking (#8), proxy endpoints/routing/mid-stream commit (#10/#12), and any change to #4's SSRF module (its primitives are reused unmodified).

## Decisions

1. **Adapters wrap #5's translate adapters; they never define a response shape (invariant 2).** `openai_compatible` delegates request serialization / response + stream parsing to #5's `openaiAdapter`; `anthropic_compatible` to `createAnthropicAdapter({ ...quirks }, { defaultMaxOutputTokens })`. The adapter's job is transport, protocol/endpoint selection, auth, JSON encode/decode, and error mapping — not normalization. Provider quirks flow to #5's quirk hooks via `config.quirks`.

2. **The HTTP seam uses #4's `createGuardedDispatcher` + `assertUrlSafe` directly — NOT `guardedFetch` — because streaming needs explicit dispatcher lifecycle (Round-1 Critical-1).** `guardedFetch` awaits `dispatcher.close()` in a `finally` before returning; `close()` is graceful and waits for the in-flight body, so an open SSE response hangs it and no body reaches `streamParse`. Instead `createGuardedHttpClient({ mode, providerKind, resolve? })` returns an `HttpClient` that: (a) `assertUrlSafe(url, opts)` (scheme/format + name-time resolved-IP gate); (b) creates a per-call guarded dispatcher (connect-time IP validation — DNS-rebinding defended at fetch time); (c) calls undici's own `fetch(url, { ...init, redirect: 'manual', dispatcher })`; (d) **rejects any 3xx** as a typed error (providers do not redirect chat/messages POSTs; this removes all cross-origin-redirect risk); (e) returns the `Response` with its body wrapped so the dispatcher is **closed when the body ends, errors, or is cancelled** (immediately if bodyless). **Dispatcher ownership is exactly-once (Round-2 High-2):** every path that fails *before* returning a body — a rejected 3xx (whose unread body is cancelled first), an `assertUrlSafe`/connect rejection, a fetch throw — closes/destroys the dispatcher itself; only once a body is handed back does the body wrapper own cleanup. So repeated provider failures cannot leak per-call agents/connections. This is correct for both non-streaming (`close` after `res.json()` drains) and streaming (`close` when the SSE stream completes/cancels). #4 is unchanged; only its exported primitives are used (and `undici` is a direct data-plane dependency, since the seam calls undici's own version-matched `fetch`).

3. **Transport steps are explicit and contract-tested (Round-1 Medium-6).** Requests set `Content-Type: application/json` and send `JSON.stringify(requestOut(ir))`; responses are `JSON.parse`d before `responseIn`. Streaming requests set `Accept: text/event-stream` and `stream: true`; the SSE reader uses **one persistent `TextDecoder({ stream: true })`** across byte chunks so a multibyte UTF-8 character split across a `Uint8Array` boundary is not corrupted, with a final flush. A local-HTTP-server contract test inspects the real headers/body; a dedicated test splits a multibyte char across chunk boundaries.

4. **Per-call context, timeouts, and cancellation are a defined contract (Round-1 High-5).** `chat`/`chatStream` accept `CallContext { signal?: AbortSignal, traceId?: string }`. A **time-to-first-byte** deadline (`config.firstByteTimeoutMs`, default 30s) aborts if no response headers / first stream event arrive; there is **no overall deadline on a stream** (streams are legitimately long) — an optional `idleTimeoutMs` may bound inter-event gaps. The caller's `signal` is composed with the timeout signal. **Caller cancellation (the caller's `signal` aborting) is breaker-neutral** — not a provider failure. There are **no automatic POST retries** (non-idempotent). A sanitized upstream request id (e.g. `x-request-id`) is preserved in error metadata for tracing; the credential never is.

5. **Two classifiers, not one: fallback eligibility vs. breaker impact (Round-1 High-4).** The error taxonomy `kind ∈ { auth, rate_limit, unavailable, bad_request, unknown_model }` feeds two separate functions:
   - `shouldFallback(kind)` — the proxy (#10) walks its chain: true for `rate_limit | unavailable | unknown_model | auth`, false for `bad_request` (a client-payload fault no other model fixes).
   - `breakerImpact(kind)` — what trips the **provider-level** breaker: `rate_limit | unavailable → trip`; `auth → trip` (persistent misconfig; the threshold absorbs a rare transient 401, half-open re-probes after a credential fix); `unknown_model → NO trip` (a retired model is model-specific, not provider-down — it must fall back without disabling every model on a healthy provider); `bad_request → NO trip`.
   Status→kind mapping refines by body where it matters: `404` → `unknown_model` only when the body/type indicates model-not-found, else `unavailable` (a wrong base path is provider-misconfig, not a missing model); `408`/`409`/`5xx`/`529`/network/timeout → `unavailable`; `429` → `rate_limit`; `401`/`403` → `auth`; `400`/`422` → `bad_request`.

6. **The breaker is a generation-versioned state machine with admission tokens (Round-1 High-3).** The pure `transition(state, event, now, cfg) → { next, decision }` (`event ∈ request|success|failure`, `decision ∈ allow|skip`, states `closed|open|half_open`) is the semantic source of truth, unit-tested exhaustively. But completion must be **correlated to the admitted generation**, or a slow request admitted while `closed` can complete after the breaker reaches `half_open` and impersonate the probe. So:
   - `before(providerId) → Admission { decision, token }` where `token = { store, generation, isProbe }`.
   - `complete(token, outcome)` with `outcome ∈ { success, trip, neutral }`, applied **conditionally on `token.generation` matching the store's current generation** (a stale completion from a superseded generation is ignored). The store increments `generation` on every `open`/`half_open` transition.
   - Half-open admits exactly **one** probe (a per-generation `probeHeld` flag); a **probe lease** (`cfg.probeLeaseMs`) lets a never-reported probe expire so the breaker isn't wedged half-open. **Reclaiming an expired lease increments `generation`** (Round-2 High-1) — so if probe A's lease expires and probe B is admitted, A's token is a stale generation and its late completion is ignored (it can't impersonate B). The lease-reclaim path gets its own test (A expires → B admitted → A completes → no effect).
   - Completions are **store-affine**: `token.store` records which store admitted the call (Redis or the in-memory fallback), and the completion goes to that same store — so a mid-call Redis outage can't transition two different stores.

7. **`withBreaker` and `withBreakerStream` map outcomes; breaker health = "did the provider respond" (Round-1 High-2, High-4).** `outcome` is derived: a resolved call or a **non-tripping** error (`bad_request`) → `success` (the provider responded → it is healthy, and a half-open probe that hits a client-fault still closes the breaker); a **tripping** error (`breakerImpact` true) → `trip`; a caller-`signal` abort → `neutral`. Because `chatStream` returns an `AsyncIterable`, `withBreaker` (which treats `fn` resolution as success) is wrong for it — a generator returns before iteration runs. `withBreakerStream` therefore owns admission across the whole iteration and derives the outcome carefully:
   - **A clean generator return is NOT automatically success (Round-2 High-3).** #5's parsers synthesize a terminal `message_stop` even when the upstream connection just closes (a truncated response over a load balancer produces no reader exception). Success therefore requires the **protocol terminal condition** — a `message_delta` bearing a `stopReason` was observed before end. An end-of-stream *without* that terminal stop is a **truncation** → `trip` as `unavailable` (with truncated-OpenAI and truncated-Anthropic tests).
   - **An observed normalized `error` event is classified, not blanket-tripped (Round-2 Medium).** Its raw `error.type` maps through the taxonomy → `breakerImpact`, so a streamed `invalid_request`/model-specific error falls back without disabling the provider, while an overloaded/5xx-style stream error trips.
   - A thrown reader/decoder error or connection reset → `trip` (`unavailable`); consumer cancellation → `neutral`.

8. **A per-instance in-memory fallback, never fail-open.** `CircuitBreaker` holds a primary `BreakerStore` (Redis) and a per-instance `InMemoryBreakerStore` fallback; a primary error routes the decision to the fallback and reports via `onError`, degrading global→per-instance breaking (mirroring the auth rate limiter) — it never drops protection (a locally-known-open provider is still skipped). The cross-instance guarantee is explicitly **best-effort under a Redis outage** (a cold fallback can't know state only another instance saw); the spec states this honestly rather than over-promising.

9. **Redis store: one atomic Lua script, Redis server clock (Round-1 High-3, Medium-8).** `RedisBreakerStore` executes each `decide`/`complete` as a single Lua script over `BreakerRedis.eval`, reading/writing `{state, failures, openedAt, generation, probeHeld}` under key `cb:{providerId}` with a TTL. It uses **Redis `TIME`** for cooldown/lease decisions (a shared clock — no cross-instance skew). Testing (Medium-8): the pure `transition` is the semantic spec; a **parity vector suite** replays event sequences through both the TS transition and the Lua-on-real-Redis and asserts identical decisions; a **concurrent `Promise.all` test** proves exactly one half-open probe is admitted. These real-Redis tests run against the repo's `docker-compose.dev.yml` Redis; they are gated on `REDIS_URL` for local runs but are **run during implementation and expected in CI** (documented), so the Lua cannot diverge silently.

10. **The four kinds get real seams; the factory rejects `local` outside self-host (Round-1 Medium-7).** `ProviderConfig` carries `kind`, `quirks?: AdapterQuirks` (passed to #5), and `extraHeaders?: Record<string,string>` (so a `subscription` or `custom` provider can add provider-specific headers without a new adapter; refreshable-credential/OAuth flows are deferred to #7). `createProviderAdapter` **rejects `kind:'local'` when `mode !== 'selfhosted'`** outright (not only via SSRF context). `subscription` is called api-key-shaped here; quota-first-then-paid ordering is #10's routing concern. **`is_free`/"marked free" is removed from this layer** — `listModels()` returns only `{ id, displayName? }`; free-marking of local/curated models is #8's catalog concern.

## Risks / Trade-offs

- [Streaming through the SSRF guard] → use `createGuardedDispatcher` + `assertUrlSafe` with body-tied dispatcher close, not the auto-closing `guardedFetch`; a real SSE test proves the first event arrives before the upstream ends and that cancellation closes the connection.
- [Concurrent half-open correctness] → generation-versioned admission tokens + single Lua transition + a concurrent probe test; the pure transition is the semantic spec and the Redis parity suite pins the Lua to it.
- [Breaker vs. fallback conflation] → separate `shouldFallback` and `breakerImpact`; `unknown_model` falls back but does not open the provider breaker.
- [Timeout kills long streams / hangs on stall] → first-byte deadline (not overall) + optional idle timeout; caller cancellation is breaker-neutral; no POST retries.
- [Credential leakage] → auth set only in a per-call headers object; errors carry status/kind/sanitized-request-id, never headers; a test greps thrown errors and a captured logger for the key.
- [Multibyte SSE corruption] → one persistent streaming `TextDecoder`; a split-boundary test.
- [Redis-outage cross-instance guarantee] → stated as best-effort local protection, not a false global promise.

## Migration Plan

Pure addition (new `providers/` module + tests + two dependency wirings: `@polyrouter/shared/server` import, `ioredis` dev type). No schema, no migration, **no change to #4's SSRF module** (only its exported primitives are consumed). Rollback = remove the module.

## Open Questions

None blocking. Refreshable/OAuth subscription credentials and provider-definition registries are deferred to #7; `subscription` quota-first ordering to #10.
