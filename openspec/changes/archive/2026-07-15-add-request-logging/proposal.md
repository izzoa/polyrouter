# Add request logging (immutable-cost RequestLog, off the hot path)

## Why

Cost tracking is the headline feature (spec §7.7), and every routed request must leave an auditable, **immutable-cost** record (spec §5 RequestLog, §7.5; invariant 4) — without slowing the request (spec §3.2.4, invariant 9) or persisting prompt/response bodies (invariant 8). #10 routes and calls; #8 gives effective-dated unit prices. This change records the outcome of each #10 request: tokens (from provider `usage`, estimated only when missing), a **snapshot of the unit prices then in effect**, the cost computed at request time, latency, the routing decision, and status — written **asynchronously, batched, and failure-isolated** so the log never sits on or breaks the request path.

The bar (spec §15): change a model's catalog price after a request completes and its recorded cost and historical spend do **not** move; a request whose provider omits `usage` is flagged `usage_estimated`, not null; a log-write failure never fails the client request.

## What Changes

- **`request_log` table + migration** (spec §5): `owner_user_id`/`org_id` (owner-FK, tenant-scoped) plus **denormalized `agent_id`/`provider_id`/`model_id` ids (NOT foreign keys)** — an append-only audit row must survive (and not fail to insert on) a concurrent provider/model/agent deletion, and keep the historical id (cost immutability). `tier_assigned`, `decision_layer`, `routing_reason`, token counts (**uncached** `input`/`output` + separate nullable cache-read/write), **`input_price_snapshot`/`output_price_snapshot`** (+ cache-rate snapshots + `price_version_id`), `usage_estimated`, `cost` (nullable = price unknown), `duration_ms`, `status`, `escalated`, `quality_signal` (both nullable — for #13/#14), `created_at`. Indexes on `created_at`, `owner_user_id`, `agent_id`, `provider_id`, `model_id`. USD-only (single-currency, per #8).
- **Pure cost/estimation** (`data-plane/src/recording/cost.ts`): `computeCost(usage, priceSnapshot)` — per-component per-1M USD, `null` when the price is unknown **or when a non-zero cache component lacks its rate** (never an understated immutable cost), distinct from `usage_estimated`; and `estimateTokens` (`chars/4` over request chars and response **text + tool name/args** — a tool-only response must not estimate 0) — **no billing tokenizer on the hot path** (invariant 9). Returns a `RequestLogDraft` metadata shape (no bodies — invariant 8).
- **Usage capture in the proxy**: #10's `ProxyCore` is extended additively so `runBuffered` exposes the response and `openStream` resolves a **stream outcome exactly once** (usage merged via #5's `mergePartialUsage` + output-char count + success/error, treating a client disconnect as error-with-partial-usage) — so streamed requests record real usage without buffering the body. `RouteDecision` gains `tierKey` so `tier_assigned` has a producer.
- **`RequestRecorder` + async `LogWriter`** (`control-plane/src/recording/`): `record()` does **no DB work** — it allocates the row `id`, runs the pure usage estimation, and enqueues a metadata draft (with the pricing *inputs* + the request-completion timestamp). The **writer** — batching **per principal** — resolves the price snapshot via #8's `resolveForModel` (with that timestamp, so cost stays immutable) under **bounded concurrency**, computes cost, and inserts via `insertMany(principal, rows)` **`ON CONFLICT (id) DO NOTHING`** (retry-idempotent) — with **bounded retry + backoff** wrapping the whole resolve-price+insert attempt (a transient DB blip doesn't create gaps), a size-bounded queue, and logged/metered drops. It flushes on interval/size and in `onApplicationShutdown` (after #10's stream drain); it never touches or fails the request path.
- **Persistence**: a `requestLogs` accessor — a batched `insertMany` (the write path) plus a tenant-scoped owned `list`/`findById` (for the cost-immutability e2e and #9-style analytics later).
- **Hook**: #10's `ProxyService` records after every completion/stream (success *and* error), fire-and-forget; the controller threads the guard's `agentId`.

## Impact

- Affected specs: **new capability `request-logging`**.
- Affected code: new `request_log` in `schema.ts` + migration; new `data-plane/src/recording/` (pure) + additive `ProxyCore` usage capture; new `control-plane/src/recording/` (recorder, writer, module) registered in `app.module`; `requestLogs` accessor in `shared/persistence.ts` + `port.ts`; `ProxyService`/controllers thread the recording context + `agentId`. Reuses #8 `resolveForModel`, #10 proxy.
- **Out of scope**: analytics queries/dashboard (§9, later); budget counters (#16); `fallback`/`escalated` status values and `quality_signal` population (#12/#13/#14 — the columns exist, unused now); prompt/response body capture (opt-in, a later change). No change to #8/#10 behavior beyond the additive usage-capture seam. Redis-backed log buffering is a documented scale option; the baseline is an in-memory batched writer (correct across instances — each appends to the shared table).
