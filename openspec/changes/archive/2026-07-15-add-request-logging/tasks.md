## 5. Recorder + async writer (control-plane)

- [x] 5.1 Add `control-plane/src/recording/log-writer.ts`: `LogWriter` — a **bounded** in-memory queue of drafts. On flush, **grouped per principal**: resolve each draft's price via #8 `resolveForModel(pricingInputs, at)` under **bounded worker concurrency** + `computeCost`, then `insertMany(principal, rows)` (`ON CONFLICT (id) DO NOTHING`). `OnModuleInit` interval flush + size-threshold early flush + **`onApplicationShutdown`** final flush (after #10's `beforeApplicationShutdown` stream drain). **Failure-isolated with bounded retry + backoff wrapping the whole resolve-price+insert attempt** (a transient failure — pricing lookup OR insert — re-queues, not immediate drop; sustained-past-budget or over-cap drops with a logged/metered count; per-principal batching so a deleted owner can't poison another tenant) — never throws to callers, never on the request path. Injectable; config for interval/size/cap/retry/concurrency.
- [x] 5.2 Add `control-plane/src/recording/request-recorder.ts`: `RequestRecorder.record(ctx, outcome)` — **allocate the row `id`**, run the pure `resolveUsage`, and build a metadata `RequestLogDraft` (id, owning principal, ids + `tierAssigned`/`decisionLayer`/`routingReason` from `RouteDecision`, `durationMs` from `startedAt`, the **pricing inputs** — model externalId/own-prices/isFree + provider baseUrl/kind — and the request-completion `at`), then **enqueue** it (NO DB call, NO price lookup on this path). Never throws into the caller. Add `recording.module.ts` (imports `DatabaseModule` + `PricingModule` for `PricingService`); register in `app.module`.
- [x] 5.3 Unit-test the recorder (fake writer): correct draft for success/error, stable id, ids/tier/decision, estimated-usage flag, no body/secret fields, never throws. Unit-test the writer: price resolution + `computeCost` (incl. immutable `at`), per-principal batching, flushes on interval/size/shutdown, a **rejecting pricing lookup AND a failing `insertMany` both retry then continue** (don't throw), **re-inserting a committed batch yields no duplicate** (ON CONFLICT), over-cap/past-budget drops counted, pricing under bounded concurrency during a stall.
