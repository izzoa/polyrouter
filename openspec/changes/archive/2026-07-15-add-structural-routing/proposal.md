# Add Layer 1 structural auto-routing (opt-in, degrades to Layer 0)

## Why

#10 accepts `model=auto` but resolves it straight to the `default` tier — a documented placeholder for "the pipeline lands in #13/#14". This change adds **Layer 1** (spec §7.1–§7.3, §7.6): cheap, sub-millisecond, **language-neutral** structural pre-classification that steers `auto` traffic to a cheaper or stronger tier. It re-introduces automatic routing *done correctly* — fixing the two failure modes that killed the reference project's rule-based classifier — without ever paying the third one:

- **System-prompt contamination (problem 1)** — score the **last user turn + recent context, excluding the system block entirely**; fingerprint the system prompt and learn a **per-agent baseline** so boilerplate identical across an agent's traffic carries zero signal (measure the *delta*, not the preamble).
- **English-only (problem 2)** — **structural** features only (effective input size, code-block presence/size, tool-definition count, tool-schema demand, multimodal presence, conversation depth, `max_tokens`); **no natural-language keyword matching** — a Japanese request with a code block scores like an English one.
- **No per-request LLM (problem 3, invariant 9)** — pure CPU arithmetic over the already-parsed IR; the hot path never runs a tokenizer or a generative classifier.

The non-negotiable bar (invariant 1, spec §7.2 guardrails): **the smart path is opt-in and always degrades to Layer 0.** `auto` is the opt-in; within it, any failure — layer disabled, Redis baseline unavailable, feature extraction throwing, no band target configured, an empty target tier — **falls back to the `default` tier and never fails or stalls the request.**

## What Changes

- **Pure structural engine** (`data-plane/src/routing/structural/`, new): `extractStructuralFeatures(ir)` + `fingerprintSystem(ir)` (de-contaminating feature extraction, no I/O) and `classifyStructural(features, baseline | null, thresholds)` → `{ band: 'high' | 'low' | 'ambiguous', score, reason }`. Language-neutral, deterministic, unit-tested. `DecisionLayer` in `resolve.ts` gains `'structural'`; `resolveTarget` is exported so a band decision reuses #12's chain-building (fallbacks apply to every layer, §7.4).
- **Band → tier mapping via RoutingRules** (`shared` + #9 reuse): `RULE_MATCH_TYPES` gains `auto_high` / `auto_low` — RoutingRules whose validated `target` (owned `tier:<key>` / `model:<id>`) designates the structural destination. They reuse #9's DTO validation + `assertTargetOwned`, are **inert in Layer 0** (the resolver only acts on `header`/`default`), and are consumed **only** by Layer 1. No migration.
- **Per-agent baseline, off the hot path** (`control-plane`, degradable): an EWMA of effective input size per `(tenant, agent, systemFingerprint)`. The **read is a synchronous in-process LRU lookup** (sub-ms, never awaits); Redis is only a **bounded, fail-fast** shared backing — a **dedicated connection with `enableOfflineQueue:false`** so a down Redis rejects immediately (no buffered commands), plus a **coalesced/throttled** background sync that drops on overflow — so the hot path does **no network I/O** and a down/slow Redis can neither stall nor fail routing (invariants 1, 9); the shared store keeps instances consistent, best-effort (invariant 10). Baselines live in a **per-`(tenant, agent)` Redis hash** with an atomic **field-count cap** + sliding TTL (hard cardinality bound, no cross-tenant `anon`), keyed by a **server-secret HMAC** of the framed system prompt (not correlatable; never recorded/logged — invariant 8).
- **Gating + tunables** (`control-plane` config, registered in the #1 fail-fast schema): `ROUTING_AUTO_LAYERS` (comma list; default `structural`; empty → pure Layer 0) plus `ROUTING_STRUCTURAL_HIGH_THRESHOLD` / `ROUTING_STRUCTURAL_LOW_THRESHOLD` / `ROUTING_STRUCTURAL_BASELINE_ALPHA` and an optional `ROUTING_STRUCTURAL_WEIGHTS` JSON override — sensible zero-tuning defaults, thresholds **and weights** exposed for power users (§7.2); the loader **fails fast unless `0 ≤ LOW < HIGH ≤ 1`** and weights are valid (bands cannot overlap).
- **Orchestration** (`StructuralRouter` in `control-plane`, wired into `ProxyService.prepare`): runs **only** when `model=auto` fell through to the Layer-0 `default` decision (so an explicit model or an `x-polyrouter-tier` header still wins — Layer 0 always wins; a confident band does override a plain `default` decision, since `auto` opted into smart routing). Extract → local baseline read → classify → fire-and-forget observe → resolve the band target (a `tier:` target carries that tier's fallback chain; a `model:` target is a single-member chain); on a high/low band with a resolvable target it **overrides** the decision with `decision_layer='structural'` and a `routing_reason` that is a **typed serialization of the band + numeric sub-scores only** (no raw text, no persisted fingerprint — invariant 8); otherwise it returns `null` and the Layer-0 `default` decision stands. The whole call is wrapped so **any** throw yields `null` (degrade).
- **Recording** (#11 reuse): a structural decision records `decision_layer='structural'` + the structured reason on the RequestLog; the served-model / fallback / cost machinery is unchanged (`request_log.decision_layer` is free text — no migration).

## Impact

- Affected specs: **new capability `structural-routing`**.
- Affected code: new pure `data-plane/src/routing/structural/` module; `'structural'` added to `DecisionLayer` + `resolveTarget` exported in `resolve.ts`; `RULE_MATCH_TYPES` extended in `shared` (auto-extends #9's `@IsIn` DTO validation); new `routing.config.ts` (env registration) + `StructuralBaselineStore` (Redis) + `StructuralRouter` in `control-plane/src/proxy/`; `ProxyService.prepare` gains the structural override step + injects `StructuralRouter`; `proxy.module.ts` provides the store/router (reuses the already-imported `RedisModule`). Reuses #9 rule CRUD, #10 resolver + proxy, #11 recording, #12 chain. **No schema migration.**
- **Out of scope (declared boundaries, not silent gaps)**: Layer 2 semantic embedding classifier + the learning loop (cloud-tier, spec §7.6/§3.3 — deferred); Layer 3 cascade escalation (#14); native `reasoning_effort` and `response_format` routing signals — both require **new `Normalized*` IR fields + round-trip parsers** (a #5 translation-contract change; bundling it here would violate "small, single-capability changes"), so this change ships the representable subset (`toolSchemaDemand` proxies structured-output demand) and the two extra sub-scores drop in additively in a fast-follow (design Decision 7); the dashboard auto-layer toggle UI (#20); any change to explicit/header/`default` routing (unchanged). §16 ToS note is unaffected (routing is provider-agnostic).
