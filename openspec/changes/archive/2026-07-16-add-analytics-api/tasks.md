## 2. Owner-scoped analytics queries

- [x] 2.1 `control-plane/database/analytics.queries.ts`: `createAnalyticsAccessor(db: Db): AnalyticsAccessor`. Every query filters `ownershipPredicate(requestLogs|requestAttempts, principal)` + `createdAt ∈ [from, to)`. **Spend = both ledgers in integer µ$ EXACTLY as `budget.reader.ts`** (`sum(round(coalesce(cost,0) * 1000000))` per ledger, so the dashboard reconciles with #16's budget counters — a float `sum(cost)` would diverge by cents), divided by `1e6` for the API's dollar figure: `summary` = one `request_log` aggregate (count, token sums, µ$ spend, `count(*) filter (where status='success'|'fallback'|'error')`, `filter (where escalated)`, `filter (where usage_estimated)`, free/paid/unpriced = `filter (where cost=0 / cost>0 / cost is null)`) + a µ$ spend over `request_attempt`, summed in JS. **Served-row semantics:** requests/tokens/split are over `request_log` rows; only spend adds attempts (documented).
- [x] 2.2 `timeseries`: `date_trunc(<unit>, created_at AT TIME ZONE 'UTC')` — **UTC-aligned** (matching #16's UTC periods, NOT the session tz) — grouped over each ledger (request metrics from `request_log`; `spend` merges the attempt-ledger µ$ sum), merged into ascending sparse points. The `<unit>` is a **fixed SQL literal chosen per validated `bucket` branch** — never interpolate raw input into SQL; everything else is a bound parameter.
- [x] 2.3 `breakdown`: `model`/`provider`/`tier` union both ledgers grouped by their own id/`tier_key` (log: `tier_assigned`); `agent` groups `request_log.agent_id` + `request_attempt` INNER JOIN its parent `request_log` for `agent_id` — with `ownershipPredicate` on **BOTH** `request_attempt` AND the joined `request_log` (attempt ownership is independent of its parent's). µ$ `spend` both ledgers, `requests` = request_log rows for the dimension; **owner-scoped** `LEFT JOIN` for `label` (providers/agents `ON id=key AND ownershipPredicate`; models through `ownedProviderIds`; model→`externalModelId`, provider/agent→`name`; tier label = key) — a deleted/foreign id → `label:null`, never another tenant's name; null dimension key → `key:''`. Order by spend desc, `limit`.
- [x] 2.4 `listRequests`: keyset — `where owner + created_at ∈ [from,to) [+ status + decision_layer + escalated] + (created_at, id) < (cursor.createdAt, cursor.id)` `order by created_at desc, id desc limit limit+1`. **Pagination:** `hasMore = rows.length > limit`; return `rows.slice(0, limit)`; `nextCursor` = the **last RETURNED** row's `(createdAt,id)` when `hasMore`, else null (NOT the limit+1th row — that would skip it). Include per-row **`attemptCostMicros`** (correlated `sum(round(cost*1e6))` of this request's attempts, 0 if none) and the **owner-scoped labels** (model/provider/agent, as in 2.3) so the safe view renders names + `total = servedCost + attemptCost` without the client re-resolving.

## 4. DTOs + service + controller

- [x] 4.1 `control-plane/src/analytics/analytics.dto.ts`: class-validated query DTOs. **DTO-level (→ 400, the global ValidationPipe default):** `from`/`to` `@IsISO8601`; `bucket` `@IsIn(['hour','day','week','month'])` (default `day`); `dimension` `@IsIn(['model','provider','agent','tier'])`; `limit` `@IsInt`+`@Min(1)`+`@Max(100)` (defaults 10/50 via `@Transform`/service); `status`/`layer` optional `@IsString`; `escalated` optional `@Transform`+`@IsBoolean`; `cursor` optional `@IsString`. **Service-level (→ 422 `UnprocessableEntityException`):** coerce `from`/`to` → `Date` and validate `from < to` + `to − from ≤ MAX_RANGE_MS` (400 days); decode/validate `cursor` (`base64 "<iso>|<id>"`).
- [x] 4.2 `control-plane/src/analytics/analytics.service.ts`: `AnalyticsService` (`@Inject(PERSISTENCE_PORT)`). Validates the range/cursor (422), calls the accessor, maps `AnalyticsRequestRow` → a **safe view** (drops `ownerUserId`/`orgId`; keeps id/createdAt/agentId/providerId/modelId/tierAssigned/decisionLayer/routingReason/status/escalated/token cols/snapshot prices/`cost` (served)/durationMs/usageEstimated/qualitySignal + `modelLabel`/`providerLabel`/`agentLabel` + `attemptCostMicros`), and returns `{ rows, nextCursor }` / summary / points / breakdown rows. Spend µ$ → dollars at the API boundary.
- [x] 4.3 `control-plane/src/analytics/analytics.controller.ts`: `@Controller('api/analytics')`, session-guarded; `GET /summary`, `/timeseries`, `/breakdown`, `/requests`, each `@CurrentPrincipal()` + `@Query()` DTO → the service.
- [x] 4.4 `control-plane/src/analytics/analytics.module.ts`: imports `DatabaseModule` (`PERSISTENCE_PORT`); provides service + controller. Register in `app.module.ts`.

## 5. Tests

- [x] 5.1 Control-plane e2e (real Postgres, stub principal guard): seed `request_log` + `request_attempt` for owners A and B across a known span. Assert: (a) **summary** totals — requests/tokens/status counts/escalated/estimated/free-paid-unpriced (over served rows), and `spend` sums both ledgers in **µ$ matching #16's rounding** (a served row + its escalation attempt sums both); (b) **timeseries** hour + day **UTC** buckets carry the right per-bucket request + both-ledger spend sums, ascending, sparse; (c) **breakdown** — model/provider/tier top-N ordered by spend with correct owner-scoped labels; the `agent` breakdown attributes an attempt's cost via its parent's `agent_id`; a null-key (no agent) row surfaces as `key:''`; a deleted-provider label is null; (d) **listRequests** — keyset pagination walks every row **exactly once** with a stable `nextCursor` (test 5 rows at `limit:2` incl. identical timestamps — no skipped/duplicated row), `status`/`layer`/`escalated` filters narrow correctly, each row carries `modelLabel`/`providerLabel`/`agentLabel` + `attemptCostMicros`, and the safe view never contains `ownerUserId`.
- [x] 5.2 **Tenant isolation**: owner B's rows never appear in owner A's summary/timeseries/breakdown/requests (counts + spend + a by-id-absent request). **Adversarial:** an A-owned `request_attempt` whose parent `request_log` is B's must NOT surface B's `agent_id`/label in A's agent breakdown (both-sides `ownershipPredicate`). **Guards:** a bad enum/ISO/limit → **400** (DTO); `from ≥ to`, an over-`MAX_RANGE` window, and a malformed `cursor` → **422** (service).
- [x] 5.3 **Index check (not a brittle EXPLAIN)**: assert `request_log_owner_created_idx` exists (`pg_indexes`) and the queries are owner+range-first (index-eligible) — a "no Seq Scan" EXPLAIN is flaky on a tiny test table, so don't assert it. Note the `statement_timeout` operational guard is deferred to config.
