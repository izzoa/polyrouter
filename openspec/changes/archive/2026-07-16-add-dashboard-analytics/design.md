# Design: add-dashboard-analytics

## Context

Reuses: #17's `/api/analytics/{summary,timeseries,breakdown,requests}` (owner-scoped, both-ledger µ$ spend, safe request rows with `modelLabel`/`providerLabel`/`agentLabel` + `attemptCostMicros` + the `decision_layer`/`routing_reason` inspector fields, keyset `nextCursor`); #18's SPA shell — the injectable `ApiClient` + `AppProvider` context seam, the `FakeApiClient` test seam, the `live={false}` prop, the store↔data boundary, and the `Range` (`24h`/`7d`/`30d`) control + providers strip (already live). The three Observe pages currently render the simulator behind a `PreviewBanner`. Governing invariants: **4** (immutable snapshot cost — render, never recompute), **1** (transparency, layer-agnostic), **8** (safe views only).

## Decision 1 — Backend: one small analytics extension (multi-value layer filter)

The dashboard's filter chips are multi-value (`explicit`→{explicit,header,default}, `auto`→{structural,cascade}). To keep them **server-side** (so keyset pagination never returns an empty filtered page while the cursor is unexhausted), extend #17's `requests`: `AnalyticsRequestsQuery.decisionLayer?: string` → `decisionLayers?: string[]`; `listRequests` filters `inArray(request_log.decision_layer, layers)` (parameterized). The DTO's `layer` accepts a comma-separated list, `@Transform`ed to a trimmed non-empty `string[]` (empty/whitespace-only → 400; each element an arbitrary non-empty layer name, bounded count/length). An e2e asserts a two-value filter + invalid input. No migration, no new backend deps.

**Dropped from scope:** an earlier plan to add per-model tokens to the breakdown is cut — "top models" ranks by **spend** (the headline), which #17's breakdown already provides; token volume is visible in the summary total + per request (table/inspector). This removes an otherwise-unused field and the ambiguity of a "by tokens" ranking.

## Decision 2 — Analytics API client + range → params

Add to `ApiClient` (`data/api.ts`): `summary(range)`, `timeseries(range, bucket)`, `breakdown(dimension, range, limit?)`, `requests(query)` — typed to #17's responses. A pure `rangeToParams(range: Range, now: number): { from, to, bucket }` maps `24h`→(now−24h, hour), `7d`→(now−7d, day), `30d`→(now−30d, day) (client clock — fine; the API just filters a range). The **`RequestRow` wire type mirrors #17's safe view EXACTLY**: `id`, `createdAt` (**ISO string**), nullable `agentId`/`providerId`/`modelId` + `agentLabel`/`providerLabel`/`modelLabel`, `tierAssigned` (nullable), `decisionLayer`, `routingReason`, `status`, `escalated`, `inputTokens`/`outputTokens`/`cacheReadTokens`/`cacheWriteTokens` (cache nullable), **all four price snapshots** `inputPriceSnapshot`/`outputPriceSnapshot`/`cacheReadPriceSnapshot`/`cacheWritePriceSnapshot` (nullable), `cost` (nullable, served), `attemptCostMicros`, `durationMs`, `usageEstimated`, `qualitySignal` (nullable). A shared `totalCostMicros(row) = round((row.cost ?? 0) × 1e6) + row.attemptCostMicros` display helper (micros-exact, so it reconciles with summary/budget spend) + `fmtMicros`, and label fallbacks `providerLabel ?? providerId ?? 'unknown'`, are the single source the table + inspector use.

## Decision 3 — uPlot chart wrapper

Add **uPlot** (`^1.6`, the §3.1 canonical lib) — the first chart dep. A small `components/Chart.tsx` SolidJS wrapper isolates uPlot's imperative lifecycle: `onMount` creates the `uPlot` instance into a ref div (series/axis/grid colors read from CSS vars via `getComputedStyle`); a `createEffect` on the reactive `data` calls `u.setData(...)`; a **second `createEffect` on `state.theme`** rebuilds/redraws so a theme toggle re-themes the chart (Overview isn't remounted on toggle); `onCleanup` **disconnects the `ResizeObserver` AND destroys** the instance; the `ResizeObserver` calls `u.setSize`. uPlot's CSS is imported once. **Scope:** the chart is **requests-per-bucket only — a single series on one scale** (avoids dual-axis/mixed-unit complexity; spend has its own card + the by-model breakdown). The x values are **epoch seconds** (uPlot's unit) produced by the tested transform. The categorical breakdowns keep the existing `BarRows` fed real data. **Testing:** rather than swallow constructor errors (which would hide prod failures), the Vitest setup provides a minimal `HTMLCanvasElement.getContext` shim so the wrapper mounts under happy-dom; the transform (timeseries → `[secs[], counts[]]`) is unit-tested as a pure function.

## Decision 4 — Overview / Costs wiring

A reusable `<RangeSelector>` (extracted from Overview's inline control) sits on **both Overview and Costs**; changing it drives `setRange`, which reloads the active page. Aggregate reads (summary/timeseries/breakdown — single-shot, no pagination) fetch on mount + on range change, plus a light `setInterval` refetch (default ~15s, gated by `live`, cleared on unmount). **Stale-response guard:** a monotonic `generation` counter is kept **per shared mutable slice** (not per page) — every loader that writes a slice (whether `loadOverview` or `loadCosts`, which share `analyticsSummary` + the breakdown slices) bumps that slice's generation and applies its response only if still current, so last-writer-wins and an old-range response can't overwrite a newer one across pages. Fetched slices (`analyticsSummary`, `analyticsSeries`, `analyticsBreakdown{Model,Provider,Agent}`, `recentRequests`) each carry loading + error state; on error the page shows a **visible error with a retry**, keeping the last-good data if any. The simulated `stats`/`chart` and the `SEED_*` overview/cost seeds are removed.

- **Overview:** stat cards from `summary` (spend, requests, tokens = `inputTokens+outputTokens`, **success / fallback / escalation** rates = `successCount`/`fallbackCount`/`escalatedCount` ÷ `requests`); `<Chart>` from `timeseries` (requests per bucket, single series); "spend by model" = `breakdown('model')` → `BarRows`; "recent requests" = a **separate `recentRequests` slice** (an unfiltered first-6 `requests` fetch — NOT the Requests page's filtered/paginated `requestList`, so it never inherits its filters or disturbs its frozen window). Empty range → a zero-state (no NaN — guard `requests===0`). Headings reflect the range (e.g. "Requests · {range}"). The providers strip stays.
- **Costs:** "Spend · {range}" + `~estimated` count from `summary`; **free / paid / unpriced** split bar from `summary.{freeRequests,paidRequests,unpricedRequests}` (percent by request count — the subscription/api sub-split stays deferred, its label says so); spend-by-model/provider/agent = three `breakdown(...)` → `BarRows`. The static price-snapshot integrity note stays. The prototype's "list price / saved" and "this month" wording are dropped (no list-price concept in #17; the range is user-selected, not the calendar month).

## Decision 5 — Requests + the reworked inspector

- **List:** the store holds `requestList` + `requestCursor` + a frozen `requestWindow` (`{from,to,filters}`) + loading/error/`generation`. `loadRequests(reset)` on reset **snapshots the current `{from,to}` + filters into `requestWindow`** and fetches page 1; "Load more" appends via `nextCursor` **reusing that frozen window** — so a later `Date.now()` can't shift the range and skip boundary rows (the no-skip requirement). The list is **not polled** (append-only log; only the aggregate pages poll), avoiding the poll-vs-pagination conflict; a `generation` guard discards a stale page. **All filters server-side** (Decision 1's multi-value `layer`): `explicit`→`layer=explicit,header,default` (deterministic routing, incl. a smart request that fell through to default), `auto`→`layer=structural,cascade`, `fallback`→`status=fallback`, `escalated`→`escalated=true`, `all`→none. The `decisionLayer` chip renders any string (unknown → neutral); the cost column shows the **total** (`totalCostMicros`) with `~` when `usageEstimated`.
- **Inspector:** reworked to the real row — header (`modelLabel ?? modelId`, id, `createdAt`); route `agentLabel → router · decisionLayer → providerLabel · tier`; **Decision** section = the `decision_layer` + the verbatim `routing_reason` string (the transparency payload — rendered whatever it is), plus `escalated` and `qualitySignal` when present; **Usage & cost** = input/output/cache tokens, the immutable price snapshots (all four; a snapshot of **`0` → "$0 free"** vs **`null` → "unpriced"/not available** — kept distinct, and cache prices can be null even on a free model), and **served `cost`, this request's attempt cost, and the total** (`totalCostMicros`, micros-exact; a null served cost is "unpriced", distinct from `$0.00`), plus the `~estimated` note when `usageEstimated`; **Timing** = `durationMs` + `status`. All values are the row's snapshots — no recompute (invariant 4). The simulator-only `steps`/`feat`/`ttfb`/`routeMs`/`protocol` rendering is removed (their data isn't recorded; acceptable for the §15 DoD = tokens/cost/latency/model/decision-layer+reason).

## Decision 6 — Test strategy

- `FakeApiClient` gains `summary`/`timeseries`/`breakdown`/`requests` returning canned, mutable data; its `requests` honors the frozen `cursor`/`status`/`escalated`/`decisionLayers` so pagination + server-side filtering are testable.
- Vitest (pure/unit): `rangeToParams` (each range → from/to/bucket); summary→derived **success/fallback/escalation** rates; timeseries→chart transform (epoch-seconds `[secs[], counts[]]`); the requests filter→param mapping (`explicit`→`layer=explicit,header,default`, etc.); `totalCostMicros` (served + attempt); the inspector view-model (decision layer/reason present, total correct).
- Vitest (component, via `AppProvider`+fake): Overview/Costs render real numbers (not seeds) + a visible error/retry on a failing load; a request row opens the inspector showing the `routing_reason`; "Load more" appends the next page **over a frozen window** with no dupes/skips; a layer filter narrows via the fake's server-side param honoring. The `<Chart>` wrapper smoke-mounts (canvas shim in the Vitest setup).
- Backend e2e: extend #17's analytics suite to assert a two-value `layer` filter (`explicit,header`) returns exactly the matching rows, and reject an empty/whitespace `layer` segment (400).

## Risks / trade-offs

- **uPlot integration surface** — imperative canvas lib in a reactive framework; bounded to one wrapper + one single-series chart, re-themed on toggle, ResizeObserver disconnected on cleanup, canvas-shimmed in tests. A wrong theme/resize is a visual bug, not a data bug.
- **All filters are server-side** — the multi-value `layer` (Decision 1) means keyset pagination never returns an empty filtered page mid-cursor. The frozen `requestWindow` + `generation` guard keep pagination stable under a moving clock / concurrent range changes.
- **Periodic refetch, not streaming** — the aggregate pages poll on a bounded interval (gated by `live`, cleared on unmount); the requests log is not polled. A true live stream is deferred.
- **New dep (uPlot) + one shared-type change (`decisionLayers`)** — no migration; the breakdown-token idea and the analytics UI's remaining polish (subscription/api split, zero-filled buckets) stay deferred.
