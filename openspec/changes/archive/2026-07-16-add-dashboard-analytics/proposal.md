# Proposal: add-dashboard-analytics

## Why

#17 built the `/api/analytics` aggregation API and #18 wired the management pages, but left the **Observe** trio — Overview, Costs, Requests — on the simulator behind a "preview — simulated" banner. This change replaces that simulator with the real API, delivering §9's dashboard-observe surface and, critically, the **routing-decision inspector** — the transparency feature (§1): every request shows its tokens, cost, latency, model, and a readable `decision_layer` + `routing_reason` (the §15 DoD). It also introduces **uPlot** (the canonical chart lib, §3.1) for the time-series chart. Both dependencies (#17 the API, #18 the SPA shell + API-client seam) are done.

## What Changes

**Backend — one small analytics extension:** the `requests` endpoint's `layer` filter accepts **multiple** values (comma-separated → `inArray(decision_layer, …)`), so the dashboard's multi-value filter chips stay **server-side** and keyset pagination never returns an empty filtered page mid-cursor. Owner-scoping, validation (400 on empty/whitespace segments), and both-ledger spend are unchanged; no migration, no new backend deps. ("Top models" ranks by spend, which #17's breakdown already provides — no per-model token extension is needed.)

**Frontend — wire the three Observe pages + the inspector to the real API:**

- **Analytics API client + range mapping.** Add `summary`/`timeseries`/`breakdown`/`requests` to the `ApiClient` (typed to #17's responses). Map the existing `24h`/`7d`/`30d` range control to `from`/`to` (relative to now) + a `bucket` (`24h`→hour, `7d`/`30d`→day). The **aggregate** pages fetch on mount + on range change (a light periodic refetch, gated by the `live={false}` test seam, replaces the simulator's 4-second feed); the Requests log is not polled.
- **Overview.** Real stat cards (spend, requests, tokens in/out, and success/fallback/**escalation** rates from the summary counts); a **uPlot** requests-over-time chart fed by `timeseries` (a small reusable single-series SolidJS `<Chart>` wrapper — create on mount, `setData`/re-theme on change, destroy + disconnect the observer on cleanup); "spend by model" via `breakdown` through the existing `BarRows`; and a real "recent requests" table (first page of `requests`). A reusable range selector drives Overview and Costs. The providers strip stays (live from #18).
- **Costs.** Range-relative spend ("Spend · {range}") + `~estimated` count from `summary`; a **free / paid / unpriced** split from the summary request counts (the subscription-vs-API sub-split stays deferred — #17); and spend-by-model/provider/agent via `breakdown` (`BarRows`). The shared range selector applies here too. The static "cost integrity / price snapshot" note stays (it describes invariant 4, which this page honors).
- **Requests + the decision inspector.** The table renders the real request rows (labels/`tierAssigned`/`decisionLayer`/tokens/**total cost**/`durationMs`/`status`), **keyset-paginated over a frozen range window** ("load more" via `nextCursor`; a `generation` guard drops stale responses), with the filter chips mapped **server-side** to `status`/`layer` (multi-value)/`escalated`. Clicking a row opens the **reworked inspector** showing the transparency fields: the `decision_layer` + `routing_reason` (rendered generically — whatever layer exists, no dependency on #13/#14 internals), the agent→router→provider route, usage & cost (input/output/cache tokens, the immutable price snapshots, served cost + this request's attempt cost + the **micros-exact total**, and the `~estimated` flag), and timing (`durationMs`, status, escalated, quality signal). Rendering reads only the row's **immutable snapshot** values (invariant 4) — never a live re-price.
- Remove the three `PreviewBanner`s and the now-dead simulated Observe slices (`stats`/`chart`/`requests` + the `SEED_*` cost/overview seeds + the `pushLiveRequest` feed). The prototype's `DecisionLayer`/`RoutedRequest` types generalize to the real shapes (arbitrary `decision_layer` strings; `status ∈ {success,fallback,error}`).

**Tests:** extend the `FakeApiClient` with the analytics methods + canned data; Vitest specs for the range→params mapping, the summary→rate/derived-value math, the timeseries→chart-series transform (pure functions), the Requests list + "load more" pagination + filter→param mapping, and the inspector rendering the real decision-layer/reason + served-plus-attempt cost. The uPlot canvas render itself isn't unit-asserted (happy-dom has no real canvas) — its data pipeline is tested as pure functions and the wrapper is smoke-mounted.

Governing invariants: **4** (render the immutable per-request snapshot cost + prices; never recompute), **1** (transparency — show whatever decision layer/reason exists, degrade gracefully; no coupling to a specific routing layer), **8** (the request rows are already secret-free safe views from #17). Out of scope / deferred: the subscription-vs-API cost split (#17), server zero-filled buckets (the UI fills), a real streaming feed (periodic refetch instead), and the Routing/Limits/Notifications config pages (#20).
