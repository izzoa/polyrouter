## 2. Backend — custom/local model pricing

- [x] 2.1 `control-plane/src/providers/models.dto.ts` (or alongside): `UpdateModelPricingDto { inputPricePer1m?: number, outputPricePer1m?: number, isFree?: boolean }` — prices `@IsNumber({ allowNaN:false, allowInfinity:false })` + `@Min(0)` + `@IsOptional()`; `isFree` `@IsBoolean()` optional. The **request-shape** rule (validated against the fields **present in the PATCH body**, NOT the merged-with-existing state) is enforced in the service: the body must be exactly one of `{ isFree:true }` or `{ inputPricePer1m, outputPricePer1m }` with **both prices present together** — so patching a single price onto an already-priced model is a 422, not a silent merge.
- [x] 2.2 `control-plane/src/providers/models.controller.ts`: `PATCH /api/models/:id` (session-guarded) → a new `ProvidersService.updateModelPricing(principal, id, dto)` (the `ModelsController` already injects `ProvidersService` — no new service); `no-store`. The service loads the model + its **owner-scoped** provider and **rejects (422) any kind other than `custom`/`local`** (model-own price is top precedence in `resolveModelPrice` — allowing it on `api_key`/`subscription` would bypass the bundled catalog). Validate the **request fields**: `{ isFree:true }` → write `{ inputPricePer1m:0, outputPricePer1m:0, isFree:true }`; both prices present → write `{ inputPricePer1m, outputPricePer1m, isFree:false }`; **anything else — a lone price, `isFree:false` alone, `isFree:true` with non-zero prices, or an empty body — is 422** (no partial/merged apply). Write via the owner-scoped `ModelAccessor.update`; 404 when not owned.
- [x] 2.3 `control-plane/src/providers/*` `SafeModel` + `toSafe`: expose `inputPricePer1m: number | null`, `outputPricePer1m: number | null` (and confirm `isFree`) so `GET /api/models` and the PATCH response carry prices for the pricing view.
