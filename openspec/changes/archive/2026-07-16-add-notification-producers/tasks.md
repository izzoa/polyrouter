- [x] 2.3 `withBreaker(breaker, providerId, fn, onOpen?)` + `withBreakerStream(..., onOpen?)`: `onOpen?: (providerId: string, info: { generation: number; openedAt: number }) => void`. After each `complete`, if `justOpened`, call `onOpen?.(providerId, …)` inside `try/catch` (never throws into the call path). In the stream wrapper, invoke from `settle` after `breaker.complete`.

## 4. Weekly spend reader (narrow, scheduler-only, built in DatabaseModule)

- [x] 4.1 `control-plane/database/`: define a `WeeklySpendReader` interface + a `WEEKLY_SPEND_READER` token; **construct the impl inside `DatabaseModule`** (it alone holds private `DRIZZLE`) and **export only the token** — do NOT add it to `PersistencePort`/`PersistenceFacilities` and do NOT re-export raw drizzle (keeps the central no-unscoped-read safeguard intact). `weeklySpendByOwner(start: Date, endExclusive: Date): Promise<{ ownerUserId: string; total: number }[]>` — sum **both** ledgers over the **half-open interval** `[start, endExclusive)` on `request_log.createdAt`: `sum(coalesce(request_log.cost,0))` grouped by `ownerUserId`, **plus** `sum(coalesce(request_attempt.cost,0))` for attempts joined to their parent `request_log` by `requestLogId` (so both ledgers share the parent's period + indexes), merged per owner. ## 5. Producers module (control-plane)

- [x] 5.1 `producers/notification-producers.ts`: `NotificationProducers` (Injectable; `NotificationService` + `REDIS_CLIENT` + config; validate threshold/window are positive ints at construction). `providerDown(providerId, name, ownerUserId)`: `void this.notifications.emit({ type:'provider_down', scope:{ ownerUserId, providerId }, fields:{ providerName: name } })` (no `lifecycleId` → dedup per provider per 1 h window). `onRequestFailed(principal)`: `bucket = Math.floor(Date.now()/windowMs)`; run **one atomic Lua** on `spike:<owner>:<bucket>` — `local n=redis.call('INCR',KEYS[1]); if n==1 then redis.call('PEXPIRE',KEYS[1],ARGV[1]) end; return n` — and if `n===threshold` `void emit({ type:'request_failures_spike', scope:{ ownerUserId: owner, lifecycleId: String(bucket) }, fields:{ count: n } })`. Whole body in try/catch → log + swallow (a Redis failure just skips the check; never throws to the proxy).
- [x] 5.2 `producers/system-mailer.ts`: `SystemMailer` (Injectable). `configured: boolean` (`resolveSystemSmtp` present). `send(to, subject, text)`: build `SmtpConfig` from `SystemSmtp` (`{host,port,secure,user?,pass?,from,to:[to]}`) → #15a `deliverSmtp(config, { title: subject, body: text }, { mode, allowedEndpoints }, timeoutMs)`. Narrow `deliverSmtp`'s `rt` param to `Pick<NotifyRuntime,'mode'|'allowedEndpoints'>` (the fields it reads). `mode`/`allowedEndpoints` from base config + `parseAllowedEndpoints`.
- [x] 5.3 `producers/weekly-summary.scheduler.ts`: `WeeklySummaryScheduler` (Injectable, `OnApplicationBootstrap` + `OnApplicationShutdown`; injects `WEEKLY_SPEND_READER`). Own `Queue('producers')` + `Worker('producers')` on dedicated `REDIS_CLIENT.duplicate(...)` connections, `'error'` listeners on both, deadline-bounded shutdown (the #15a patterns). Bootstrap reconciliation is **fail-open** (deadline-bounded + caught + retried in the background — a down Redis must NEVER delay/reject boot, invariant 1): if `NOTIFY_WEEKLY_ENABLED` `queue.upsertJobScheduler('weekly-summary', { pattern: NOTIFY_WEEKLY_CRON, tz:'UTC' }, { name:'weekly_summary' })` else `queue.removeJobScheduler('weekly-summary')`. The fail-open reconciliation uses a **single** background backoff loop (no overlapping `upsertJobScheduler` calls, unref'd timer, cancelled on shutdown). Worker `weekly_summary` (**at-least-once**, idempotent): read the scheduled occurrence from `job.opts.prevMillis` (NOT `job.timestamp` = creation time) → `endExclusive = new Date(prevMillis)`, `start = new Date(prevMillis - 7*86400000)`, `occurrenceKey = String(prevMillis)`; `const rows = await reader.weeklySpendByOwner(start, endExclusive); for (const r of rows) await notifications.emit({ type:'weekly_spend_summary', scope:{ ownerUserId: r.ownerUserId, lifecycleId: occurrenceKey }, fields:{ total: formatMoney(r.total) } })` — **await** the emits (occurrence-keyed `lifecycleId` + `emit` dedup collapse a stalled re-run to one summary/owner/week).
- [x] 5.5 `notifications/notification.types.ts` (#15a renderer): change the `weekly_spend_summary` body wording from "Total spend this week" to **"Known spend this week"** (null-price rows coalesce to 0, so an all-unknown owner isn't shown a misleading total).
- [x] 5.4 `producers/producers.module.ts`: imports `DatabaseModule` (for the exported `WEEKLY_SPEND_READER`) + `RedisModule` + `NotificationsModule`; providers `PRODUCERS_CONFIG`, `NotificationProducers`, `SystemMailer`, `WeeklySummaryScheduler`; exports `NotificationProducers` + `SystemMailer`. Side-effect import `producers.config.ts`. Register in `app.module.ts`.

## 7. Tests

- [x] 7.1 Unit (data-plane) — `breaker*.spec.ts`: `complete` returns `justOpened:true` exactly on closed→open (threshold trip) and half_open→open (failed probe); `false` on success/neutral/non-final trips and when already open; the `CircuitBreaker` zeros `justOpened` for a **fallback-store** completion and on a Redis-store fault. **Lua all-paths parity**: stale-token and neutral completions return the tuple (not scalar 0) and in-memory/Lua agree on `justOpened`/`generation`/`openedAt`. `withBreaker`/`withBreakerStream` call `onOpen` once on the opening completion, never on others, and a throwing `onOpen` never surfaces to the caller.
- [x] 7.2 Unit (control-plane) — `notification-producers.spec.ts`: `providerDown` emits the right scope/fields, never throws; `onRequestFailed` (fake ioredis) emits only when the counter **reaches** the threshold, sets a window `lifecycleId`, pexpires on the first incr (atomic Lua), and swallows a Redis/emit failure. `system-mailer.spec.ts`: `configured` reflects config; `send` builds the SmtpConfig + calls `deliverSmtp`; unconfigured is guarded. Auth: `sendResetPassword` returns **synchronously** (a never-resolving mailer doesn't block), skips+warns when unconfigured, never logs token/url (canary), swallows a mailer rejection.
- [x] 7.2b (control-plane) proxy: with a fake `NotificationProducers`, a chain that trips the breaker calls `providerDown` once with the provider name/owner; a throwing/down producer never fails the request. Scheduler: with a **blackholed Redis**, `WeeklySummaryScheduler` bootstrap **resolves without throwing** (fail-open, boot not gated); a repeated `upsertJobScheduler` yields one scheduler and disable removes it.
- [x] 7.3 Control-plane e2e (real Postgres + Redis; #15a stub SMTP + a channel wired to it): (a) **provider_down** — a provider whose calls fail, tripped via repeated proxy calls, delivers one `provider_down` naming it, request still completes; a re-trip in-window doesn't re-alert. (b) **request_failures_spike** — drive `NOTIFY_FAILURE_THRESHOLD` failed requests for an owner → one spike per window; below-threshold → none; a second owner's failures don't count toward the first (owner-scoped counter); a **post-commit stream** error and a **cascade** error each feed the counter. (c) **weekly_spend_summary** — seed `request_log` **and** `request_attempt` for two owners across in/out of the `[start,end)` window, invoke the worker handler directly, assert each owner gets only their own total **including cascade-attempt cost** and excluding out-of-window rows; an **all-null-cost** owner renders "Known spend this week: 0" (not a crash, not a misleading total) (cross-tenant + both-ledger + bounded interval + null-cost). (d) **password-reset** — stub SMTP as server-wide SMTP, trigger a reset → the stub receives the mail, the reset call returns promptly, **no token/url in captured logs**; unconfigured → request succeeds + warn, nothing sent. (e) **isolation** — a blackholed notify queue leaves a proxy request and a reset request succeeding.
