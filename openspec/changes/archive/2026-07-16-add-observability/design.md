# Design: add-observability

## Context

The proxy path today: the three `/v1` controllers are guarded by `AgentApiKeyGuard` (controller-scoped `@UseGuards`); `ProxyService.prepare()` resolves the route (Layer 0 + structural/cascade); the data-plane chain runners drive the attempts as **lazy closures** built in `buildBundle` — the provider adapter is constructed per attempt in `buildAdapter` (where the full provider row, including `name`, is in scope) and may fail during setup (missing credential, decrypt failure) before any upstream call; `RequestRecorder.record()` synchronously enqueues a metadata-only draft; `LogWriter` batches drafts per principal, resolves the price snapshot, computes `cost` (`computeCost`, USD or `null` for unpriced) **rebuilding rows on each insert retry**, and drops rows after bounded retries when the DB stays down. Streamed upstream failures are not always throws: adapters can yield a normalized `error` event, and the breaker treats a stream that ends without a terminal event as a failure. `configureApp` (in `app.setup.ts`) is shared by `main.ts` and every e2e harness. `SessionGuard` early-returns for non-`/api` paths. The circuit breaker (data-plane) has states `closed|open|half_open`; the chain options currently expose only an `onOpen` (transition-into-open) listener.

Spec §3.2.6 requires OTel traces (spans auth → scorer → upstream call → DB write) and Prometheus metrics attributing latency/errors per provider and per routing layer; §14.10 schedules it now. TODOS #21 additionally requires circuit-breaker **state** metrics.

## Goals / Non-Goals

**Goals:**
- One trace per authenticated proxied inference request (`/v1/chat/completions`, `/v1/messages`): root `proxy.request` with children `auth` → `routing` → `upstream` (one per attempted chain member, streaming included) → `recording.enqueue`; requests that exit early (auth failure, budget block, route error) produce a partial chain; the **actual DB write** is traced at the writer as a batch span linked to the originating requests.
- A Prometheus `/metrics` endpoint: request/latency/token/cost per provider, model, and decision layer; per-provider upstream error attribution (including setup failures and non-throw stream failures); breaker **state** gauge + open transitions; log-writer drops; process metrics.
- Observability can never change a request outcome; hot path stays cheap (invariant 9); metadata only (invariant 8).
- Data-plane: one additive, behavior-preserving seam (breaker-state observation callback); nothing else.

**Non-Goals:**
- OTel metrics/logs signals; auto-instrumentation bundles; HTTP/pg/redis monkey-patching.
- Tracing the control-plane CRUD/dashboard API; counting pre-routing rejections in `requests_total` (auth failures are visible via the `auth` span and 401s; budget blocks via their 4xx — normatively excluded from the request counter, which counts recorder-finalized inference requests).
- Per-tenant metric labels or dashboards (cloud-tier, §3.5); alerting; a metrics UI in the SPA.

## Decisions

1. **Manual spans via `@opentelemetry/api`; SDK registered only when enabled.** Code always calls `trace.getTracer('polyrouter')` — a no-op unless a provider is registered. `main.ts` calls `initTracing()` (new `observability/tracing.ts`) immediately after `loadConfig`, before `NestFactory.create`: when `OTEL_ENABLED`, it registers a `NodeTracerProvider` (resource `service.name` = `OTEL_SERVICE_NAME`) with `BatchSpanProcessor(OTLPTraceExporter)`; `provider.register()` installs the ALS context manager. A **malformed** `OTEL_EXPORTER_OTLP_ENDPOINT` with tracing enabled fails boot fast (registered as an optional URL in the config fragment, §12 discipline); an absent endpoint uses the exporter's standard default; a well-formed but **unreachable** collector is non-fatal forever (batched async export, failures contained). An `OnApplicationShutdown` provider awaits `shutdownTracing()` (flush + shutdown, idempotent). Tests never call `initTracing`; the tracing e2e registers its own provider with an `InMemorySpanExporter` through the same global API.
2. **Root span in an Express middleware mounted in `configureApp`** (`app.use('/v1', otelRootMiddleware)`) — every e2e harness inherits the wiring. It starts `proxy.request` (attrs from `req.method` + `req.originalUrl` — mounted middleware sees a stripped `req.path` — + client protocol openai|anthropic derived from the path), wraps `next()` in `context.with(trace.setSpan(context.active(), span), next)` so ALS carries the context through guards/services/stream pumps, and ends the span on `res.once('close')` with `http.response.status_code` (double-end guarded; covers streaming and client aborts). `/v1/models` and early-exit requests simply produce shorter chains. A recording enqueue that settles just after close may end its child after the root — legal in OTel, accepted.
3. **Child spans at the request phases.** `auth`: inside `AgentApiKeyGuard` around verification (outcome attr; no key/prefix/hash material; exceptions propagate unchanged). `routing`: around `ProxyService.prepare()` (attrs post-resolution: `decision_layer`, `tier`, `model`, `cascade`; route errors mark the span error and rethrow). `upstream`: per attempted chain member (decision 4). `recording.enqueue`: around `RequestRecorder.record()` — explicitly the request-path enqueue, NOT the durable write.
4. **The durable write is traced at the writer (spec's "DB write").** When tracing is active, drafts carry the originating request's `SpanContext` (non-persisted field, captured at enqueue). Each writer batch insert (`writeGroup`/`writeAttemptGroup`) runs under a `recording.write` span with **span links** to the drafts' request contexts (attrs: row count, principal-batch label-free; status error + a `dropped` attr on the give-up path). Links — not parenting — because one batch serves many requests; this keeps batching intact and makes insert failures/drops visible in traces.
5. **Upstream observation wraps the adapter, built lazily in `buildAdapter`.** `buildAdapter` (which has the provider row) returns `observeAdapter(this.factory({...}), { provider: provider.name })` — construction stays lazy inside breaker admission. The decorator wraps:
   - `chat`: span + timer around the promise; throw ⇒ error outcome + rethrow.
   - `chatStream`: the wrapper is itself an async generator delegating to the inner one, so the span starts **on first iteration** (body entry — a never-iterated call creates no span) and ends exactly once in `finally` (completion, upstream throw, or consumer abort via `return()`), mirroring the breaker's stream-failure rule: a yielded `error` event or ending without a terminal event ⇒ outcome `error`; consumer abort before completion ⇒ outcome `canceled` (span OK + attr — cancellations must not inflate provider error rates); clean terminal ⇒ `success`.
   - `listModels`/`testConnection` pass through untouched.
   The decorator emits `upstream_requests_total{provider,model,outcome}` + `upstream_duration_seconds{provider,model}` on the same boundary (model = `request.model` at call time). **Setup failures** (adapter never built: credential/decrypt errors) are disjoint from decorator coverage and are counted separately in `buildAdapter`'s failure path as `upstream_setup_failures_total{provider}` — no double count is possible, and per-provider error attribution covers both phases.
6. **Metrics: `prom-client` with a per-Nest-app `Registry`** (never the global default — Jest builds many apps per process). A `ProxyMetrics` injectable owns the registry, `collectDefaultMetrics`, and exception-safe emit methods. Series (prefix `polyrouter_`):
   - `requests_total{protocol,decision_layer,status}` + `request_duration_seconds{protocol,decision_layer,status}` — emitted from `RequestRecorder.record()` (enqueue-time, so traffic stays visible during DB outages). **Normative:** counts recorder-finalized inference requests, exactly once each (cascade attempt-ledger rows do NOT increment it). The `protocol` label is required — `RecordingContext` gains `protocol: ClientProtocol`.
   - `tokens_total{provider,model,direction}` — from `record()` AND `recordAttempt()` (a superseded cascade cheap call consumed real provider tokens; billing counts it, so do metrics).
   - `cost_microusd_total{provider,model}` — emitted by the writer **exactly once per draft, only after its batch insert succeeds** (retries rebuild rows but must not re-emit; a dropped row emits no cost — the drop counter covers the loss). Unit: `round(costUsd × 1_000_000)`; `null` (unpriced) rows emit nothing (they remain visible via requests/tokens and the row's `usage_estimated` handling). The counter mirrors the authoritative snapshot cost, never a recomputation (invariant 4).
   - `upstream_requests_total` / `upstream_duration_seconds` / `upstream_setup_failures_total` — decision 5.
   - `breaker_state{provider}` gauge (0 closed / 1 half_open / 2 open) + `breaker_opens_total{provider}` — decision 7.
   - `log_rows_dropped_total` — every row the writer abandons (insert give-up and any queue-cap shedding share this counter).
   Labels are bounded by instance config (provider display names, external model ids, layer/status/outcome enums) — never tenant/agent/request ids or content. Provider names are tenant-chosen config strings visible to whoever can scrape — metadata (invariant 8), noted on the endpoint.
7. **Breaker state via one additive data-plane seam.** The chain-runner options (alongside the existing `onOpen`) gain an optional `onBreakerState?: (providerId: string, state: BreakerState) => void`, invoked with the state observed at each admission decision (allow, probe, or skip). The proxy supplies it per request (like `onOpenFor`) mapping providerId → provider name and setting the `breaker_state` gauge; `onOpen` additionally increments `breaker_opens_total`. Semantics documented on the metric: last-observed state per provider on THIS instance at traffic time (stale without traffic; shared-Redis truth converges at the next attempt). No behavior change; existing callers unaffected; data-plane unit coverage for the new callback.
8. **Provider-name/protocol pass-through.** `RecordingContext`, `RequestLogDraft`, and `RequestAttemptDraft` gain `providerName: string`; `RecordingContext` gains `protocol`; drafts gain the optional `spanContext`. None are persisted — DB row shape unchanged.
9. **`GET /metrics`**: returns `registry.metrics()` with prom-client's content type; `NotFoundException` when `METRICS_ENABLED=false`. Unauthenticated by design (SessionGuard ignores non-`/api`) — instance-level, metadata-only aggregates, same trust posture as `/health`; `BIND_ADDRESS` defaults to loopback; #22's compose docs will note network-guarding.
10. **Module wiring.** New `ObservabilityModule` (providers `ProxyMetrics` + shutdown hook; controller `MetricsController`; exports `ProxyMetrics`) imported by `AppModule`, `RecordingModule`, and `ProxyModule`; it imports nothing (no cycles). Within one Nest app all importers resolve the same `ProxyMetrics` instance.

## Risks / Trade-offs

- **OTel SDK 2.x API drift** (constructor-injected processors, renamed `ReadableSpan` parent accessors). Mitigation: pin `@opentelemetry/api` 1.9.x + one tested stable 2.x SDK release; only `tracing.ts` and the tracing e2e touch SDK types. `sdk-trace-base` is a devDependency because tests import `InMemorySpanExporter` directly.
- **Global tracer state in tests**: `provider.register()` is global per module registry; Jest isolates modules per file and e2e runs `--runInBand`. The tracing e2e is the only registrar and shuts down in `afterAll`; the disabled-default posture is exercised by every other suite rather than by a same-process "second app" (a registered global provider cannot be unregistered).
- **Streaming span lifetime**: spans start on first iteration and close in `finally`; a hung consumer keeps a span open bounded by the existing first-byte/idle timeouts. A never-iterated stream call produces no span — accepted (no upstream work happened).
- **Breaker gauge staleness**: state is observed at traffic time only; an idle instance shows the last-seen state. Accepted and documented — a scrape-time Redis sweep would add I/O and a provider enumeration the hot path doesn't need.
- **Metrics cardinality** bounded by config size, not traffic; a hosted multi-tenant instance would revisit labels (cloud-tier, out of scope).
- **Always-on metrics cost**: label lookups + counter increments per request (sub-µs, no I/O) — invariant 9 holds; tracing when enabled adds batched async export only.
- **`/metrics` is unauthenticated**: usage patterns (models, spend rates, provider names) visible to whoever reaches the port. Accepted for self-host parity with `/health` + the env kill-switch; flagged for #22 docs.
