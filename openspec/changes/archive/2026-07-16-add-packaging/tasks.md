Compose-owned env: `BIND_ADDRESS=0.0.0.0`, `NODE_ENV=production`, `MODE=selfhosted`, `DATABASE_URL=postgresql://polyrouter:${POSTGRES_PASSWORD}@postgres:5432/polyrouter`, `REDIS_URL=redis://redis:6379`, `BETTER_AUTH_URL=${APP_URL:-http://localhost:3001}`, the four secrets interpolated from `.env`; optional vars (`METRICS_ENABLED`, `OTEL_ENABLED`, `OTEL_EXPORTER_OTLP_ENDPOINT`, `APPRISE_API_URL`, `NOTIFY_ALLOWED_ENDPOINTS`, `NOTIFY_APPRISE_EGRESS_CONFIRMED`, OAuth ids/secrets) as **bare keys** (unset stays unset — several reject empty strings; `${VAR:-}` would break boot).

- [x] 6.2 Drain proof with a LIVE stream: exec a loopback SSE stub inside the app container (`kind: local` provider — the self-host loopback SSRF exception), mint an agent key via the API, hold a slow streamed completion open, `docker stop` the app — assert chunks were FLOWING and the client alive at stop time, the stream terminates cleanly (`[DONE]` received — the drained-stream evidence; the app logs nothing distinctive at drain, so the terminator + exit state ARE the proof), and the container's **inspected** `.State.ExitCode` is 0 / `.State.OOMKilled` false (compose's own exit code proves nothing). Restart → healthy again, data intact (idempotent migrations).
