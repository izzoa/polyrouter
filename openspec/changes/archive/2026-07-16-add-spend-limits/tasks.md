## 3. Redis SpendCounter (µ$) + period math

- [x] 3.1 `budgets/period.ts`: pure `periodInfo(window, at: Date): { periodId; startMs; endMs }` — UTC calendar period id (`YYYY-MM-DD` / `YYYY-Www` ISO / `YYYY-MM`) + UTC start/end epoch ms. Unit-tested at boundaries. `toMicros(usd: number): number = Math.round(usd*1e6)`.
- [x] 3.2 `budgets/spend-counter.ts`: `SpendCounter` (Injectable). Constructor builds a **dedicated** `REDIS_CLIENT.duplicate({ enableOfflineQueue:false, commandTimeout: BUDGET_REDIS_TIMEOUT_MS, maxRetriesPerRequest:1 })` (error handler, eager connect, disconnect on shutdown). `key(owner, scope, scopeId, window, periodId)`. `read(keys[]): Promise<number[]>` → `MGET` (missing=0) — the block-check read. `reconcileMax(key, micros, ttlMs): Promise<number>` (scheduler write) → atomic Lua `local c=tonumber(redis.call('GET',KEYS[1]) or '0'); local v=tonumber(ARGV[1]); if v>c then redis.call('SET',KEYS[1],v) end; redis.call('PEXPIRE',KEYS[1],ARGV[2]); return math.max(c,v)` — **monotonic** so an out-of-order/older occurrence's `SET` can't lower the counter (safe: single-writer + append-only period ledger ⇒ a later snapshot ≥ an earlier one). `heartbeatSet(nowMs, ttlMs)`/`heartbeatAgeMs(nowMs): number` on a fixed `budget:reconcile:heartbeat` key. `markOnce(key, ttlMs): Promise<boolean>` → `SET NX PX`. Each op relies on the connection's `commandTimeout`; callers treat a throw as "unavailable". (Background scheduler traffic may use a second duplicate connection at scale — noted.)

## 4. Budget cache + service (block path)

- [x] 4.1 `budgets/budget-cache.ts`: `BudgetCache` (Injectable; `PERSISTENCE_PORT`, config). `get(principal): Promise<BudgetRow[]>` — **capped** (`BUDGET_CACHE_MAX`, LRU-ish) per-owner in-memory cache with `BUDGET_CACHE_TTL_MS` freshness and **single-flight** (one in-flight load per owner); a load error returns last-known or rethrows-as-unavailable per fail mode. `invalidate(owner)` (called by CRUD).
- [x] 4.2 `budgets/budget-service.ts`: `BudgetService` (Injectable; `BudgetCache`, `SpendCounter`, `NotificationProducers`, config). `applies(b, agentId)`; `keyFor(owner, b, now)` (its `(scope,scopeId,window,periodId)` key). `BudgetHit = { budget: BudgetRow; spentMicros: number; periodId: string; resetAt: Date }`. `checkBlocked(principal, agentId): Promise<BudgetHit | null>` — cache → filter `enabled && action==='block' && applies`; if none, return null. Else check the reconciliation **freshness**: `SpendCounter.heartbeatAgeMs > BUDGET_STALE_MS` (or absent) ⇒ **enforcement-unavailable** (a stopped scheduler mustn't read counters as 0 and silently allow). Otherwise `SpendCounter.read` the **distinct** keys once, then check **every** matched budget against its own key's value + `toMicros(amount)`, return a `BudgetHit` for the first over-threshold budget (its spent/period/reset). **On enforcement-unavailable or any Redis/cache fault**: `BUDGET_FAIL_OPEN` → return null (allow), else throw a sentinel the controller maps to `503 budget_enforcement_unavailable`. `notifyBlocked(principal, hit): void` — `markOnce('budget-blocked:'+hit.budget.id+':'+hit.periodId)` then `void producers.budgetBlock({ ..., spent: hit.spentMicros, channelIds })`; swallow.

## 6. Budget scheduler (own queue: reconcile = sole writer + alert)

- [x] 6.1 `budgets/budget.scheduler.ts`: `BudgetScheduler` mirroring `WeeklySummaryScheduler` but on its **own** queue `QUEUE_NAME='budget-eval'` (NOT the shared `producers` queue), `SCHEDULER_ID='budget-eval'`/`JOB_NAME='budget_eval'`; fail-open bootstrap, conditional worker (`BUDGET_SCHED_ENABLED`), bounded shutdown; injects `BUDGET_READER`, `SpendCounter`, `NotificationProducers`, config. Extracted pure `runBudgetOccurrence(reader, counter, producers, at)` where `at = new Date(prevMillis - 1)` (so a run at exactly a boundary evaluates the **just-closed** period). **Group `listActiveBudgets()` by distinct counter key** (`owner|scope|scopeId|window|periodId` via `periodInfo(window, at)`); per key: `micros = await spendMicrosFor(owner, scope==='agent'?scopeId:null, new Date(startMs), new Date(endMs))`; **`await SpendCounter.reconcileMax(key, micros, endMs-at.getTime()+grace)`** (monotonic — an out-of-order occurrence can't lower it); then for each `action='alert'` budget on that key with `micros >= toMicros(amount)`, if `await markOnce('budget-alerted:'+id+':'+periodId, ttl)` → `await producers.budgetAlert({ ownerUserId, agentId?, budgetId:id, periodId, name, spent: micros, threshold: toMicros(amount), channelIds })`. **After processing all keys, `await SpendCounter.heartbeatSet(now, BUDGET_STALE_MS + grace)`** (proves reconciliation is alive; the block check treats a stale/absent heartbeat as enforcement-unavailable). Await the emits.

- [x] 11.2 Control-plane e2e (real Postgres + Redis): (a) **CRUD + tenant isolation** — CRUD a budget; cross-tenant by-id → 404; agent-scope w/o agentId → 422; a check-constraint violation is rejected. (b) **block across instances** — seed `request_log` over a `block` budget, run the occurrence (reconciles the counter + heartbeat), assert `checkBlocked` returns a `BudgetHit` and the proxy path 402s with **no upstream call**; a second owner/agent unaffected (scoping); the same counter is seen from a second `SpendCounter` (shared Redis, no per-instance drift). (c) **fail mode** — a down/bounded Redis: `BUDGET_FAIL_OPEN=true` allows within the deadline; `=false` → 503. **Stale/absent reconciliation heartbeat** (healthy Redis, no scheduler run): a `block` budget with counters present-but-stale (or absent) routes through the fail mode too — `=false` → 503 (not silently allowed as under-budget), `=true` → allowed. (d) **alert once/period** — over-threshold alert budget: run the occurrence twice → one `budget_alert`; different period → alerts again; `channelIds` targeting delivers only to the selected channel. (e) **reconcile** — seed `request_log` + `request_attempt` (incl. an agent-scoped attempt), clear the counter, run the occurrence → counter equals both-ledger sum; window boundary consistent. (f) **non-blocking** — a down notify queue leaves a blocked request rejected promptly + an allowed request succeeding.
